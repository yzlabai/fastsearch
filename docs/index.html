Built for retrieval that cites its sources

It takes the structure-extraction fast path — interpreting PDF content streams for coordinates instead of rendering pages to pixels — so output is byte-identical and parses are sub-10 ms.

🦀

One pure-Rust binary

~29 MB, zero runtime deps, <10 ms warm parse (~700 pages/s). Drop it on a server, a laptop, or a CI box — no environment to set up.

📍

RAG-native citations

Every chunk carries page + bbox + heading breadcrumbs. Reverse locate(x, y) lookup, with 100% coverage — answers that point back to the exact spot on the page.

🔌

Four faces, one output

CLI · library · MCP (stdio) · REST — and the output is byte-identical across all four. Wire it straight into an agent.

🔍

In-process OCR

--ocr runs ONNX on tract (PP-OCRv6 tiny by default). Digital pages never touch a model; CCITT G3/G4 fax + JBIG2 scans covered. ~7 MB fetched on first use.

🧠

Embedded models, opt-in

Merged-cell table structure, formula → LaTeX, full-page transcription (UniRec-0.1B), plus PP-DocLayoutV2 / DocLayout-YOLO layout. None of it baked into the binary.

🛡️

Safe by default

Hidden-text filtering (flagged & auditable, never silently dropped), zip-bomb & page-count guards, per-page complexity profiling. The deterministic core stands alone.

bash
# build once — no toolchain zoo, just cargo
$ cargo build --release

# full IR: provenance + coordinates
$ docparse input.pdf -f json

# clean Markdown
$ docparse input.pdf -f markdown

# RAG chunks — page + bbox + breadcrumbs
$ docparse input.pdf -f chunks

# OCR scans (free for digital pages)
$ docparse scan.pdf --ocr

A core that owes nothing to the AI

A Cargo workspace of 17 crates. The deterministic core stands alone; neural models attach only where they earn their keep.

🏗️ Deterministic by design

  • core depends on no PDF library — reading order & output are format-agnostic; adding a format is one trait + one registry line.
  • A self-built PDF content-stream interpreter — a graphics/text matrix state machine emitting positioned chunks.
  • An independent font layer (ToUnicode CMap / AFM / Encoding), with veraPDF as the algorithmic reference only — no copied code.
  • Per-page rayon parallelism; identical input → byte-identical output.

🧩 Models at the boundary

  • Neural models never enter the core — they attach per page through an Enhancer boundary.
  • Only a hard page routed to a model is ever rendered — on demand, in pure Rust.
  • Every model output carries a source tag + capped confidence, so provenance is always traceable.
  • All optional models are Apache-2.0, fetched as external files — never baked into the binary.
