## 背景与缺口

B6 让向量召回在 PG 跑 ANN（pgvector 直查），但 ANN 需要 `embedding` 列有值。此前 `apply_upsert`（CDC
落地路径）只把嵌入写进**引擎侧派生索引**（`VectorStore`），**不写 PG `embedding` 列** → pgvector 直查
模式下 CDC 投递的 chunk 永远没有向量、直查命中不了。缺口 = "CDC 自动写穿 PG embedding"。

## 关键设计权衡：CDC 写穿反馈环

写穿的难点不是写，而是**反馈环**：consumer 从复制槽消费 chunk → 嵌入 → `set_embedding` UPDATE PG
→ 该 UPDATE 又进 WAL/复制流 → consumer 再消费 → 再嵌入 → 再写穿 …… **无限环**。这正是该项被推迟的原因。

**断环方案（双防线）**：

1. **列清单 publication（主防线，结构性）**：publication 改为**只发布源真源列**（`COLUMNS` 的 17 列），
   **排除引擎派生/记账列** `embedding`/`embed_model`/`updated_at`。写穿 UPDATE 只动这三列 → **PG 不为其
   产生复制事件** → 环从根断开。语义上正确：`embedding` 是**派生**，本就不该进"源真源"复制流（守不变量
   #2 PG 真源/引擎派生）。代价：列清单发布需 **PG 15+**（核心特性、非扩展，不破不变量 #1 托管 PG 可移植；
   RDS/Supabase/Neon 均 15+）。
2. **`set_embedding` 幂等守卫（副防线，行为性）**：`WHERE ... AND (embedding IS DISTINCT FROM $1 OR
   embed_model IS DISTINCT FROM $5)` → 值未变即 **0 行更新**，PG 对 0 行 UPDATE 不产生复制事件。即便某
   部署的 publication 未排除派生列，也能阻尼环（重复 apply 同一向量不再二次写）。两防线互为冗余。

## 怎么实现

- **pg（`set_embedding`/`clear_embedding`/DDL）**：`set_embedding` 加 `model` 参 → 写 `embed_model` + 刷
  `updated_at` + 幂等守卫；新 `clear_embedding`（空文本媒资清向量，仅 `embedding` 非空时更新）。DDL
  publication 改列清单（`COLUMNS`），**幂等且自愈不抢占**：无→`CREATE`（包 `EXCEPTION WHEN
  duplicate_object OR unique_violation` 防并发首建 TOCTOU）；本表 legacy 全列发布（`pg_publication_rel.prattrs
  IS NULL`）→ `ALTER SET TABLE` 收敛到列清单；属别表的同名 publication 不动（避免多表互抢）。
- **engine（`apply_upsert` 路由）**：配了 `set_pg_vector` → 写穿 PG（`block_on_pg` = block_in_place 桥，
  与 B6 检索同约束，需 multi-thread runtime），**不写引擎派生索引**（直查模式读路径不用它）；空文本 →
  `clear_embedding`。未配 `vector_pg` → 维持原"写本地派生索引"。新增 `embed_model` 字段 + `set_embed_model`。
- **server**：pgvector 模式装配时 `set_embed_model("{model}@{dim}")`（落 PG embed_model，溯源 + 幂等）。

## 怎么验证（Docker pgvector pg16 真机，DATABASE_URL gated）

- **pg `b6_set_embedding_idempotent_guard`**：首次写穿 1 行、相同向量+模型重写 **0 行**（断环行为基元）、
  换 `embed_model` 1 行、`clear_embedding` 幂等（清 1 行→再清 0 行）。
- **engine `b6_cdc_write_through_to_pg_embedding`**（multi-thread）：`apply_upsert` 写穿 → 同 embedder 嵌
  query 直查**命中 chunk 1、引用 page 来自 PG**；文本变空 → 写穿路径清向量 → 直查不再命中。
- **结构性**：纯函数单测 `ddl_has_extension_table_publication` 断言列清单 publication 排除 `embedding`/
  `embed_model`/`updated_at`、含源列。
- **不回归**：**CDC 闭环 4 测试全绿**（`cdc_closed_loop_pg_to_search`/`cdc_embed_hybrid_full_loop`/
  `cdc_consume_persist_crashsafe`/`cdc_initial_snapshot_bootstrap`）——证明列清单 publication 不破常规
  逻辑解码。收口三绿（fmt/clippy -D warnings/`cargo test --workspace` 含 DATABASE_URL 全 gated 套件）。

## 过程中修掉的真问题

- **并发首建 publication TOCTOU**：新增的并行 PG 集成测试触发——多个 `ensure_schema` 同时见"publication
  无"→ 都 `CREATE` → 一个报 `unique_violation`(23505, `pg_publication_pubname_index`)。修复：`CREATE` 包
  `EXCEPTION WHEN duplicate_object OR unique_violation THEN NULL`（注意并发竞态在 PG 表现为 `unique_violation`
  而非 `duplicate_object`——两者都接）。3× 并行重跑稳定 31/31。
- **多表互抢同名 publication**：先前把 `ALTER SET TABLE` 无条件执行 → 并行不同表的测试互相把 publication
  重指到自己 → 催生竞态。改为**仅 legacy 全列发布时才 ALTER**（`prattrs IS NULL`），属别表则不动。
