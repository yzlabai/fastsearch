## 背景

审查的安全向核查确认了一个重要正面结论：**所有检索入口的 ACL 强制注入无遗漏、object token 绑定扎实**。R3 处理其余 5 处安全问题（M19-M22 + XSS/5xx），全为纯 Rust 本机可收口。每项先补失败测试并负控确认再修。

## 做了什么

- **M19 `/v1/similar` 种子未做 ACL 校验**（跨租户存在性/相似性预言机）：`more_like_this` 改为先 `stored_row_by_gid` 取种子行做 `AclFilter::visible` 校验——种子对调用者不可见时**当作不存在返回空**，绝不以其机密正文构造 MLT 查询、也不泄露存在性（对齐 `resolve_citation`）。测试：租户 B 用租户 A 的机密 chunk 当种子 → 空；A 自己 → 正常召回。
- **M20 HMAC 签名串分隔符歧义**：`mac_hex` 从 `format!("{cid}|{exp}|{ct}")` 改**长度前缀 framing**（每字段前置 8B BE 长度）。`cid`（含客户端可控 doc_id）/`ct`（客户端 media_type）可含 `|`，旧拼接下 `(cid1,ct1)` 与某 `(cid2,ct2)` 能拼出逐字节相等的消息 → 签名跨对复用。framing 后字段边界唯一可解析。测试：构造旧方案会碰撞的两对，断言签名不跨对复用。
- **M21 限流桶无界 + 限流在鉴权前**（开启限流反引入未认证内存 DoS）：① 把 6 个 Bearer 端点的 `allow` 移到 `principal_from_headers`/`require_principal` **之后**——伪造 token 在鉴权处被拒、不创建桶，桶数 ≤ 配置 key 数（有界）；② `RateLimiter::check` 加硬上限 `MAX_RATE_BUCKETS`，触顶且来新 key 时整表 `clear()`（摊还 O(1)，避免"触顶后每请求 O(n) 逐出"），作 token-gated 端点（无 principal）的 backstop。测试：灌 MAX+5000 个不同 key，断言 map 有界。
- **M22 集合注册表非租户隔离**（任意认证用户可覆盖/读取/枚举他人集合）：注册表 key 从 `String` 改 `(Option<String>, String)`=`(owner_tenant, name)`，`create`/`get`/`list`/index-dim 校验四处全部按调用者 tenant 分桶。`tenant=None`（管理 key）自成作用域；单租户部署行为不变。测试：两租户各注册同名 `kb` 不互相覆盖、GET 只见自己的、list 不枚举他租户。
- **R3.5 同源存储型 XSS + 5xx 泄露**：① `serve_inline_bytes` 加 `is_safe_inline_ct` 白名单（图片/音视频/PDF）——非白名单类型（含客户端可伪造的 `text/html`、`image/svg+xml`）降为 `application/octet-stream` + `Content-Disposition: attachment` + `X-Content-Type-Options: nosniff`，不内联渲染；白名单媒资照常内联（保 `<img>/<video>`）。② `internal_err` 助手：PG 错误（`pg upsert`/`pg delete`）完整记服务端日志、只回客户端通用文案，不泄露表名/SQL 片段。

## 取舍

- **M21 clear() vs 精细逐出**：初版用"逐出 idle 满桶"的 retain，但触顶后每个新 key 都 O(n) 全扫 → CPU DoS 隐患。改整表 `clear()`（摊还 O(1)，仿动态数组倍增；清空只把桶重置为满、不放松长期限流）。极端下所有人被重置为满桶，可接受。
- **R3.5 白名单 vs nosniff-only**：单加 nosniff 挡不住"攻击者声明 text/html"（nosniff 让浏览器**遵从**声明类型 → 仍渲染 HTML）。真正的修法是 MIME 白名单：非白名单强制下载。这保留了 inline `<img>` 主用例（MM6-signer 的设计目标），又堵住 HTML/SVG 脚本。text/plain 未入白名单（会下载），罕见且更安全。
- **5xx 脱敏范围**：只脱敏了明确泄露 schema/SQL 的 PG 站点（review 点名的"表名/SQL 片段"）；引擎/嵌入错误是 typed `EngineError`、敏感度低，本轮未动（诚实记账，可后续统一）。
