## 1. PG 集成测试真机跑 → 发现并修并发 boot 竞态

起 Docker pgvector（`wal_level=logical`），`DATABASE_URL` 下 `cargo test -p fastsearch-pg`：
**并发跑 4/16 非确定失败**（再跑换另外 4 个失败 → 确认是并发竞态非脏数据）。根因：引擎
多副本/无状态，多副本 boot 都跑 `ensure_schema` 的幂等 DDL，`CREATE EXTENSION/TABLE/INDEX/
PUBLICATION ... IF NOT EXISTS` 在 Postgres 仍有 **TOCTOU 竞态窗口**。

修复（[fastsearch-pg/src/lib.rs](../../crates/fastsearch-pg/src/lib.rs) `ensure_schema`）：整段
DDL 包进事务 + `pg_advisory_xact_lock(固定 key)` 串行化，锁随 COMMIT/ROLLBACK 自动释放。
**这是生产加固**（多副本并发 boot 安全），不只是测试问题。+1 回归测试
`ensure_schema_concurrent_no_race`（8 连接并发全成功）；全 17 集成测试并发多轮全绿。
另：全工作区 `cargo test --workspace` 带 `DATABASE_URL` = 224 测试全绿、零失败。

## 2. Ollama 真语义嵌入端到端（B4 续：真模型 + 跨语言）

起 server `FASTSEARCH_EMBEDDER=ollama FASTSEARCH_EMBED_MODEL=nomic-embed-text-v2-moe
FASTSEARCH_EMBED_DIM=768`，index 3 个语义不相干的中文 chunk（猫/量子计算/股市），`mode=vector`：

| 查询（英文，与中文 chunk **零字符重叠**） | 命中 #0 | 判定 |
|---|---|---|
| quantum superposition and entanglement for parallel computation | 量子计算 chunk | ✅ 跨语言语义 |
| interest rates and investor sentiment moving equity prices | 股票市场 chunk | ✅ 跨语言语义 |

英文 query 召回对应中文 chunk 排第一 → 证明 `HttpEmbedder(Ollama)` → engine 嵌入 passage →
向量索引 → 向量检索全链路在**真多语言模型**下工作（非 HashEmbedder 基线）。这条路径此前
只有确定性基线单测，**首次真模型端到端验证**（含跨语言）。
