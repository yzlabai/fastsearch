## 背景

全仓审查（6 路并行深读 + 真机复现）出 40+ 处发现，共性是"**测试全绿掩盖了边界输入下的正确性破洞**"——多处功能表面存在、实际反向劣化。本轮做 R1（P0 正确性急救，7 高危）+ R2（P1 检索语义一致，9 中危），全为纯 Rust 本机可收口。**每项先补能复现问题的失败测试并做负控确认，再修**（多数问题正因缺该测试而长期潜伏）。

## R1 — 正确性急救（7 高危）

- **H1 rerank 反向破坏排序**（两条链路）：① 向量独有命中在 rerank 时 `text_map` 无对应 → 拿空串 → 全候选 0 分退化 gid 序：回真源 `stored_text(gid)` 补正文（engine）。② `LexicalOverlap::tokenize` 对 CJK 整句成单 token（`is_alphanumeric` 对中文为 true）→ 中文候选恒 0：改字符 bigram 切分（rerank，无外部分词依赖）。
- **H2 Float 过滤漏召回**（core+text 两端）：core `Eq/Ne/In` 从变体严格 `PartialEq` 改数值互比（`Int(6)==Float(6.0)`），并要求字段存在（缺失→false，含 Ne，修契约含糊）；text `field_as_u64` 去掉 Float 截断分支（`Lt(page,5.5)` 截成 `<5` 漏 page=5）→ Float 值退 AllQuery 交精确后过滤。
- **H4 单请求 OOM abort**：`SearchRequest::validate` 给 `top_k`/`candidates` 加上界（`Vec::with_capacity` 巨值 → `handle_alloc_error` 不可 unwind → 打崩多租户服务），`k*8` 改 `saturating_mul`。
- **H5 HNSW 墓碑聚簇返 0**：删一簇相似向量后其墓碑占满 `k×over_fetch` 窗口，无 filter/acl 时不升级 → 返 0。`needs_escalation` 拆出 `use_predicate`（分支选择）与"有墓碑也升级"；升级上限改**图节点总数**；关键：无 filter/acl 时**走纯 `search`**（不走 `search_filter`——查询落在被删簇里时 filtered-traversal 会被困住），再加**暴力精确安全网**（HNSW 欠交付则 `brute_search` 兜底，硬正确性保证）。
- **H6 Python Retriever 真客户端必崩**：`FastsearchClient.search` 加 `highlight` 参数透传进 body；`_FakeClient` stub 改为**镜像真实签名**（去 `**kw`，否则吞参遮蔽崩溃）；补 `inspect.signature.bind` 直接回归（不依赖 live server）。
- **H7 README.zh-CN 上手命令失效**：`--data ./idx` 时代命令 → server + REST 客户端命令（CLI 已改纯客户端）。

## R2 — 检索语义一致（9 中危）

- **M1 空 And 归零**：`{"and":[]}` core 语义恒真，但 Tantivy 空 `BooleanQuery` 是 `EmptyScorer` → 整检索归零。`translate`/`exact_translate` 空 And → `AllQuery`（后者供 `Not(And([]))` 补集正确）。
- **M2 time 过滤两端不同构**：text 侧 `StoredRow::get` 缺 `time_start_ms`/`time_end_ms` → keyword 召回整支归零；从 `media.time` 补齐（与 vector 侧 `VecMeta::get` 同口径）。
- **M3 text over-fetch 无升级**：不可翻译谓词（HeadingPrefix/Exists）退 AllQuery，高频词下窗口被后过滤筛到 < k。重构成"单轮 `run` 闭包 + 翻倍升级循环"（对齐 HNSW 安全网），条件"够 k / 窗口未填满 / 覆盖全库"收敛。
- **M5 rerank 窗口失效**：`RerankSpec.top_k` 全 workspace 无处读取。rerank 前按融合序 `truncate(spec.top_k)`（hits 此时已是融合序）。
- **M6 分组键缺 collection**：`auto_merge` 键补 `collection`，`collapse_groups` 键补 `collection`（doc_id 档）/ `collection+doc_id`（section_id 档，section 是 doc 内编号）。用 NUL 分隔。
- **M7 空 query 图检索污染**：Hybrid（serde 默认）+ query_image + 空 query 时 keyword 走 match-all 污染融合。先算 `query_vec`，`image_only`（空 query + 有向量）时跳过 keyword → Hybrid 结果等于 Vector。
- **M8 非跨模态图查询混比**：`embed_query_image` 除 `caps().image` 再校验 `cross_modal`（写入侧从不把图向量入共享文本索引 → 否则图向量与文本向量无意义余弦），拒绝而非静默返垃圾。
- **M9 text tie-break 用 DocAddress**：改为**构建窗口内全部通过后过滤的候选 → 按 (分, GlobalId) 排序 → 截 k**，跨索引重建返回序稳定（守不变量 #4）。
- **M4 旧快照 modality 漏召回**：`modality` `serde(default)` → 旧快照 ""；`MemVectorIndex::load` / `HnswVectorIndex::load` 若空由 `kind` 回填（同 text 侧派生口径）。

## 取舍

- **H5 安全网 vs 纯升级**：先试"墓碑也走 search_filter"，实测被墓碑 clique 困住仍返 0（5 次重跑才复现稳定），改"纯 search + 暴力兜底"——`brute_search` 只在 HNSW 欠交付时触发（罕见），代价可控且给硬保证。
- **H2 简化选项**：Float 值一律退 AllQuery（不做 `ceil/floor` 边界翻译）——page/section 是整数字段，Float 仅 JSON 产物，牺牲一点预过滤选择性换正确性，正确性由后过滤保证。
- **M9 性能取舍**：改为读全窗口候选（不早停）才能按 gid 稳定排序，doc-store 读增加；正常查询窗口小（k×4），仅低选择性升级时变大。热路径读优化归 R4.4（诚实记账）。
- **M8/H4 行为变更**：非跨模态 query_image 从"静默空/垃圾"变"报错"；超大 candidates 从"放行"变"400"——都是把静默错误变显式失败，更安全。
