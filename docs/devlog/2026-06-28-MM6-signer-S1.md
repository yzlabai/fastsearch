# devlog 2026-06-28 — MM6-signer S1：AssetFetch::InlineRef + fetch_inline_bytes

> 上游：[MM6-signer 资产网关签名 URL 设计](../plans/2026-06-28-MM6-signer-资产网关签名URL设计.md) 里程碑 S1。本环境+gated 模式（字节路 Docker PG 验证）。

## 做什么（S1：基础重构，行为保持）

为后续"签发短时 URL 让前端 `<img src>` 取图"（S2/S3）铺地基：让 `resolve_citation` **只定位、不取字节**，字节改由独立的已授权出口按需取。

- engine：`AssetFetch::InlineBytes(Vec<u8>)` → **`InlineRef`**（无字节，仅表"该 cid 有 inline 字节可取、已过 ACL"）。`resolve_citation` 的 `Inline` 分支：有 `source_pg` → `InlineRef`（**不再 block_in_place 取字节**，省一次 PG 读）；无 → None。
- engine 新增 **`fetch_inline_bytes(cid) -> Result<Option<Vec<u8>>>`**：**不带 acl**（调用方须已授权——authed 网关先过 resolve 的 ACL；S2 的 token 端点已验签）；从 `source_pg` 真源 `block_in_place` 直查；无 source_pg / cid 非法 / 无字节 → None。doc 强标"仅供已授权出口内部调用，勿新挂别的入口"。
- 消费方分流（保持行为）：server `/v1/asset` 的 `InlineRef` 臂 → 持锁调 `fetch_inline_bytes(cid)` 吐字节 + `media_type`（来自 resolve）/ None→404；MCP `resolve_citation` 工具 `InlineRef` → `{kind:"inline_ref"}`（指针，字节经 REST 取）。
- 删除死变体 `InlineBytes`（resolve 不再产出）。

## 为什么这样设计（review 要点）

- **省一次 PG 读 + 解耦**：旧实现 resolve 即取字节；多数调用（MCP 给指针、未来签 URL）并不需要字节本身。InlineRef 把"定位"与"取字节"分开，resolve 零字节。
- **fetch_inline_bytes 无 acl 是 by-design**：authed 网关已在 resolve 处过 ACL；S2 token 即"已授权"凭证（presigned 语义）。强标注 + 仅两出口调用，守不变量 #3。
- **media_type**：字节面只返字节；authed 端 Content-Type 取自 resolve 的 `ResolvedAsset.media_type`；S2 token 端将把 `ct` 编入签名（免 PG 再查），故 fetch_inline_bytes 不需返 media_type。

## 怎么验证

- 本环境 `fetch_inline_bytes_without_source_pg_is_none`：无 source_pg → None；非法 cid → None。
- Docker `mm6_inline_serves_bytes_from_source_pg` 更新：授权 → `InlineRef` + `fetch_inline_bytes` 吐真源字节、越权 None（待 Docker 运行验证）。
- server Docker `asset_inline_bytes_e2e`：行为不变（端点内部改走 fetch_inline_bytes）。
- 收口三绿：fmt + clippy --workspace -D warnings + test --workspace（30 套件）。
