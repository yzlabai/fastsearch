# devlog 2026-07-22 — FHT 结构化旋转（替物化 d×d，Step 1+2）

> 上游：[FHT plan](../plans/2026-07-22-FHT结构化旋转.md)、[MAX_DIM 决策记录 §6](../governance/2026-07-21-向量旋转维度上限与DoS.md)（"根治"方向）。spec 回写 [15-vector](../specs/15-vector.md) §6。

## 做什么 / 为什么

把 TurboQuant 的数据无关旋转从**物化 d×d 高斯矩阵 + Gram-Schmidt**（存 O(d²)、建 O(d³)、
apply O(d²)、标量无 SIMD）换成 **FHT 结构化旋转**（随机化 Walsh-Hadamard）：存 **O(d)**、
apply **O(d·log d)**、零建矩阵。这是 MAX_DIM 决策记录点名的"根治"——同时消掉 DoS 巨分配和高维标量瓶颈。

| d | 物化 apply/向量 | FHT | 物化存储 | FHT 存储 |
|---|---|---|---|---|
| 1536 | 2.36M | **0.023M（~100×↓）** | 9.4 MB | ~8 KB |
| 3072 | 9.44M | **0.049M（~190×↓）** | 38 MB | ~16 KB |

**关键**：turbovec 自身用物化 d×d（靠 BLAS 快），fastsearch 上 FHT 在**纯 Rust 无 BLAS**约束下反超它。

## 怎么做

**Step 1 原语 `fht.rs`（test-only 先验证）**：`StructuredRotation` = 零填充到 `D=next_pow2(d)` →
`ROUNDS=3` 轮 `[±1 符号翻转（固定种子）→ 原地归一化 WHT（O(D log D) 蝶形）]`。
- **正交 → 保内积**：`H/√D` 与 ±1 对角均正交 → `⟨R(q),R(v)⟩=⟨q,v⟩`（零填充不改内积）→ 量化估计器/
  精排语义**不变**。**确定**（固定种子）、**纯 Rust 无 unsafe/无依赖**。
- +7 测：保内积（含非 2 幂 768/1536，<5e-3 漂移）/保范数/能量摊匀/确定/d=4096 无巨分配/WHT 对拍 H₄/
  out_dim 填充。

**Step 2 接 turbo（无条件替换 + v2）**：
- `TurboVectorIndex` 用 `StructuredRotation` 替物化 `Rotation`；`packed_len`/`query_lut`/`score`
  全按 **D=`out_dim()`**；`search` 用 `q_rot.len()` 作打分维；快照存**原始 d**、D 派生。
- **决策：无条件替换**（非 opt-in flag）——turbo 新档（1 天、无生产数据），FHT 严格更优**且实测召回
  不劣**，故不引 `fht:true` plumbing。**快照 v1→v2**（码空间 d→D 不兼容，v1 load 拒之）。
- **物化 `Rotation` 保留给 `BruteBinaryRotated`**（未动）；`ensure_rotation` 因 FHT 无巨分配 → revert
  非 Result；`mod fht` 转无条件。

**review 跟进（上一提交复审小注）**：`MemVectorIndex::upsert` 旋转档**先验证维度后赋 dim**（避免超维
首 upsert 毒化 dim），同 turbo；+断言。

## 怎么验证

- **FHT 端到端召回门禁** `recall_vs_exact_fht`（d=1000→D=1024 走填充，聚簇 n=1500）：4-bit 纯量化分
  exact@10 **0.885** vs 精确暴力——**与物化档 ~0.87 同量级/略优**，即证 FHT 召回不劣、无条件替换成立。
- 既有 turbo 全测（持久化往返 v2/确定/维度校验/排序/预过滤/ACL/毒丸 corr/超维拒）在 FHT 下**全绿回归**；
  engine `persist_reopen_restores_turboquant_backend` 回环（v2 save→load）绿。
- **收口三绿**：fmt + clippy `-D warnings` + `cargo test --workspace` **315 passed / 0 failed**
  （vector 61→**69**：+7 FHT 原语 +1 recall；quant/turbo 回归）。

## 取舍 / 诚实记账

- **填充膨胀码**：非 2 幂维 `D>d` → 码宽按 D（1536→2048、3072→4096 = **+33% 码字节**；2 幂维 0%）。
  换 apply ~100–200×↓、存储 MB→KB——净大赢。
- **turbo 快照不向后兼容**（v1→v2）：无生产 v1 数据，load 对 v1 显式报错（非静默）。
- **BruteBinaryRotated 仍物化档**：本次只切 turbo；Step 3 再评估切它 + 放宽 MAX_DIM。
- FHT 3 轮 HD 的各向同性略欠满随机旋转，但实测召回不劣 → 够用；若未来某数据不达标，可 turbo v3 加
  flag 回退物化（当前实测无需）。

## Step 3（同日续作）：BruteBinaryRotated 切 FHT + 退役物化旋转

把最后一个物化旋转档 `MemVectorIndex`/`BruteBinaryRotated` 也切 FHT，并**删除物化 `binary::Rotation`**
（+ `next_unit`），FHT 成**唯一**旋转实现：

- `MemVectorIndex.rotation` → `fht::StructuredRotation`；code/l1 转 D 空间（`pack_signs`/`l1_norm` 按 D），
  **精排仍用原 d 向量**（正交保内积，两阶段语义不变）。**无格式变更**（该档快照存 f32、code/l1 派生，
  旋转 load 后由 `set_rabitq_rotation` 重算）——不像 turbo 需 v2 bump。
- `binary::Rotation`/`next_unit`/`MAX_ROTATION_DIM` 删除/迁移：cap 移到 `fht::MAX_DIM`（含义从"d×d DoS
  闸"变"sanity/`next_pow2` 溢出界"，8192 不变）；`ensure_rotation`/`set_rabitq_rotation` **revert 非
  Result**（FHT 无巨分配，MAX_DIM 显式早检即够）——这把上一轮 MAX_DIM 硬化的 fallible-Rotation 机制
  优雅收掉（FHT 消掉了它守的 d×d 分配）。
- **MAX_DIM 未放宽**（保 8192）：真实嵌入 ≤4096 已覆盖，为不存在的 >8192 模型放宽属 Speculative
  Generality（code smell），不做。
- **验证**：`rabitq_rotation_helps_anisotropic`（FHT）rot **0.925** vs norot 0.775（+15pt，过 +5pt 门禁）——
  略低于物化 0.97（3 轮 HD 各向同性稍欠），但强度足够、精排兜底正确性。**实测 ROUNDS 扫描**：
  3→turbo 0.885/aniso 0.925；4→0.870/0.933；5→0.865/0.942——**ROUNDS=3 turbo 最优**（主用例），
  多轮拿 aniso 换 turbo 不值 → 定 3。全覆盖 oversample=精确、确定性等既有测全绿回归。
- **收口**：fmt + clippy `-D warnings` + `cargo test --workspace` **314 绿**（vector 68：删 `rotation_dim_guard`
  −1）。
