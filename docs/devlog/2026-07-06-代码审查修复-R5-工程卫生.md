## 做了什么

- **M23 SDK 静默丢弃 collection**：两个 SDK 的 `search` 都没把 `collection` 放进请求体（Python 不注入、TS `void collection`），多集合 server 上会跨集合返回命中。修复：与 CLI 一致注入 `{"eq":["collection", name]}` 过滤，与用户 filter `and` 合并。**测试**：TS 16 + Python 6 全绿（含"注入单 Eq / 与用户 filter and 合并"断言）。
- **M26 CLI 无请求超时**：`ureq::post()` 用默认 agent（无读超时），server 挂起则 CLI 永久阻塞。改为 `Client::new` 建带 `timeout` 的 `Agent`（连接/读/写，默认 30s，`FASTSEARCH_TIMEOUT_SECS` 可配）。
- **R5.4 低危清理**：① text `search(k=0)` 返空（M9 的 build-all→truncate(k) 已结构性修正，补 lock 测试）；② 删 `Filter::and_acl` 死代码（零调用 + 误导：Filter AST 无法表达多值 ACL）；③ `set_embedding`/`vector_search` 加 `ensure_finite` 守卫（NaN/inf 向量提前清晰报错，不让 pgvector 晦涩拒收）；④ 删 `fastsearch-sync` 未用依赖 `bytes`/`futures-util`。
- **R5.5 卫生 + 文档回写**：① `git rm --cached` 两个已被跟踪的 `.pyc`（.gitignore 规则加在提交之后）；② CLAUDE.md:68/71 回写 vector（HNSW/量化/filtered-traversal/墓碑压实/H5 安全网已落地）、sync（pgoutput 解码 + H3 重取 + M16 上限已落地，仅流式为后续）；③ deploy/README 更正基镜 `1.85→1.88`（对齐 Dockerfile）+ 澄清"已部署 test ns"（手工）vs"待运行验证"（自动化 build/CI）不矛盾。
- **M25 Docker/CI**：① 新增 `.dockerignore`（排除 `target/`（本地实测 32GB）、`node_modules`、`vendor/docparse/target`、data/idx、docs 等）；② 修 Dockerfile 虚假的"依赖预热缓存层"注释（实际无独立依赖层，如实说明 + 指向 cargo-chef/dummy-src 方案）；③ CI 加 **sdk** job（TS `npm test` + Python `test_integrations.py`——此前 clients/ 完全无 CI，正是 H6 崩溃盲区来源）+ **docker-build** job（发布制品此前从未被 CI 构建）；④ hot-path-isolation 断言补 `fastsearch-mcp`（检索入口）+ `fastsearch-cli`（默认档），本机 `cargo tree` 确认二者零 docparse。
