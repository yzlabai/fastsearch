## 背景与取舍

看板"RaBitQ/量化"下一迭代项。**完整 RaBitQ**（随机旋转 + 1-bit + **无偏内积估计器**）实现精巧、估计器

估计器作其上的精化、下一迭代。

## 做了什么

- **`binary.rs` 原语**：`pack_signs(归一化向量) → Vec<u64>`（每维符号 1 bit，按 u64 字打包）；
  `hamming(a,b) = Σ popcount(a^b)`。+3 单测（pack/hamming、跨字、0.0 视正号）。
- **`MemVectorIndex` 两阶段粗筛（opt-in）**：新 `with_binary_prefilter(oversample)`；`Entry` 加 `code`
  （upsert/load 时由归一化向量派生，不落盘）。search：开启时 → **Hamming 粗筛 top-`k·oversample`**
  （键 `(hamming 升, gid 升)`，确定候选集）→ 仅对候选做 **f32 精确重排**；关闭时（默认 `None`）= 原精确暴力。
  filter/ACL 仍在粗筛**之前**施加（守不变量 #5）；重排 + GlobalId tie-break → 确定。

## 为什么正确 + 快

- **正确**：符号 Hamming 越小 ⇒ 符号一致维越多 ⇒ 余弦越高（**粗代理**）；候选集内用 f32 精确重排 → 最终
  top-k 在候选集内精确。vs 全局精确的 recall 由 oversample 决定。
- **快**：粗筛每条 ~`d/64` 次 u64 `xor+popcount`（d=384 → 6 字操作）vs 精确 `d` 次浮点乘加 → 粗筛阶段
  快一两个数量级；只对 `k·oversample`（远小于 N）条做 f32 重排。

## 怎么验证（本环境，纯 Rust，确定）

+5 单测（`cargo test -p fastsearch-vector`，22 全绿）：
- **`binary_full_oversample_equals_exact`**（强保证、无统计抖动）：oversample 大到覆盖全集 → 候选=全部 →
  结果与精确暴力**逐条 chunk_id 相同 + 分值 f32 精确**。证明重排路径正确。
- **`binary_recall_high_with_moderate_oversample`**：oversample=8、7 个查询，**recall@10 ≥ 0.85**。
- **`binary_is_filter_aware`**：二值档 + `kind=table` 过滤 → 只返回 table（粗筛前置过滤，守 #5）。
- `pack_and_hamming_basics` / `pack_spans_multiple_words`：原语正确性。

收口三绿（fmt/clippy -D warnings/`cargo test --workspace`）。**默认 `None`=精确暴力 → 既有 22 套件零回归**
（exact 路径逐字节不变）。

## 后端化（续，2026-06-27）

把库级 primitive 接成引擎/server **可选后端**，使部署可启用。

### 做了什么

- **`VectorBackendKind::BruteBinary(usize)`** 新档（`usize`=oversample）+ 常量 `DEFAULT_BINARY_OVERSAMPLE=8`。
- **`VectorStore` 接线**（无新 enum 变体——二值是 `Brute(MemVectorIndex)` 内的属性）：`new(BruteBinary(m))`
  → `Brute(with_binary_prefilter(m))`；`kind_str` 据 `MemVectorIndex::binary_oversample()` 返回
  `"brute_binary"`/`"brute"`；`load(BruteBinary(m))` 载入同样 f32 快照再 `set_binary_prefilter(Some(m))`
  翻档（**与 brute 共享 on-disk 格式**，二值只是检索策略）。
- **engine `open_with`**：检查点 `vector_backend="brute_binary"` → 恢复 `BruteBinary(默认 oversample)`
  （**记格式、调参取默认**，与 HNSW 参数同策略——line 296 既有先例）。
- **server**：`FASTSEARCH_VECTOR_BACKEND=brute_binary` + `FASTSEARCH_BINARY_OVERSAMPLE`（默认 8）。

### 验证

- **vector `vectorstore_brute_binary_roundtrip`**：`kind_str="brute_binary"`、落盘 + `load(BruteBinary)`
  重载仍粗筛档、结果逐条一致。
- **engine `persist_reopen_restores_brute_binary_backend`**：首启 `BruteBinary` → persist 记
  `brute_binary` → **重开即便传默认 `Brute` 也恢复粗筛档**（检查点覆盖默认）、检索可用。
- **server 实跑**：`FASTSEARCH_VECTOR_BACKEND=brute_binary FASTSEARCH_BINARY_OVERSAMPLE=6` boot → health 200。
- 收口三绿（含全 workspace）。**默认仍 brute 精确，零回归**。

### review

- **持久化一致性**：二值与 brute on-disk 格式相同（f32 快照），`kind_str` 区分重载档；oversample 取默认
  （非数据、属调参，同 HNSW），不丢格式、不误载。✓
- **确定性**：BruteBinary 仍走 `MemVectorIndex` 的确定二阶段（f32 重排 + gid tie-break）。✓
- **诚实记账**：完整 RaBitQ（旋转 + 无偏估计器）仍标下一迭代；本次只把已验证的 BQ 粗筛接成可选后端。
