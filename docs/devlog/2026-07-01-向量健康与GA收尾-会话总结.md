## 做了什么（按提交）

**① 组（本机可收口，全部完成）：**
- **HNSW 墓碑自动压实**：`compact`/`maybe_compact`（墓碑过半且超阈值时活集原地重建图，摊还 O(1)），守不变量 #6 墓碑增长项。+2 测试。
- **inline 字节 HTTP Range**：`parse_range`+`serve_inline_bytes`（单段 `bytes=A-B`/`A-`/`-N`→206/416/`Accept-Ranges`；多段退 200），token 验签仍在取字节前。+6 测试。回写 spec 19-server v1.9。
- **CI 依赖许可门禁**：`deny.toml`（allow-list 经 **cargo-deny 0.19.9 本机实跑** `licenses ok, sources ok`）+ `about.toml`/`about.hbs`（**cargo-about 0.9.0 实跑**产 THIRD-PARTY-NOTICES）+ parse-ocr/parse-tables 编译档。审计全树无唯一 copyleft/share-alike。
- **HNSW 真·图内 filtered-traversal**：复核发现 hnsw_rs 0.3.4 已有 `search_filter`/`FilterT`（原 spec"需上游支持"判断有误）→ filter+ACL 谓词下推进遍历。+1 测试。
- **1.1 RaBitQ 旋转**：复核已端到端接线，仅修正 binary.rs/lib.rs 陈旧注释。

**② 组（真机验证 + 加固）：**
- **fix：`ensure_schema` 并发 boot 安全**——引擎多副本 boot 都跑幂等 DDL，`CREATE EXTENSION/TABLE/INDEX/PUBLICATION IF NOT EXISTS` 仍有 TOCTOU 竞态（实测 `DATABASE_URL` 下并发 4/16 非确定失败）→ 事务级 `pg_advisory_xact_lock` 串行化 DDL。+1 回归测试；Docker pgvector 真机 17 测试并发多轮全绿。**这是生产 bug，非仅测试问题。**
- **2.2 pgvector 直查**：复核已实现（B6），本次真机验证（发现上面的并发 bug）。
- **2.3 embedding 写穿**：复核大半已实现；**Ollama 真模型 e2e**——`nomic-embed-text-v2-moe` 768d，英文 query **跨语言**召回中文 chunk 排第一（量子/股市，零字符重叠），证明语义嵌入全链路真模型工作。
- **2.1 向量对标出数**：自包含 harness `benchmarks/vector_bench.py`（合成高斯簇 + numpy 暴力真值，离线可复现）+ [RESULTS.md](../../benchmarks/RESULTS.md)。实测（release+HNSW，单线程）：fastsearch recall@10≈**0.99**（vs Qdrant 1.0），延迟 **~5–7ms vs ~1ms**——`ef_search` 对 fastsearch 几乎无影响 → 每查询固定开销（Mutex 串行 + 融合管线 + 命中组装）主导、非 ANN。**过程含两个 harness bug（同 doc_id 触发 doc 级替换、`k` 字段应为 `top_k`）与 debug/release 教训（53ms→5ms）**，均已修正记录。

**MM8b 选型调研（唯一硬阻塞项）：**
- deep-research + 一手来源复核 → [选型调研](../plans/2026-07-01-MM8b图像检索模型选型调研.md)：主选 **jina-embeddings-v4**（备选 SigLIP 2 / Qwen3-VL-Embedding-2B），**经 vLLM 走 OpenAI 兼容 `/v1/embeddings`**。**关键修正：Ollama 无法嵌图像（#7677），MM8b 走 OpenAI 路径非 Ollama**。
