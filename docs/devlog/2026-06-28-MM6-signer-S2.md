# devlog 2026-06-28 — MM6-signer S2：HMAC token + token 门控字节端点

> 上游：[MM6-signer 设计](../plans/2026-06-28-MM6-signer-资产网关签名URL设计.md) 里程碑 S2；承接 [S1](2026-06-28-MM6-signer-S1.md)。本环境完整可验。

## 做什么

让前端 `<img src>` **免 Bearer 头**取 inline 图字节——靠短时 HMAC 签名 URL（presigned 语义）。

- **`AssetSigner`**（server）：`sign(cid,ct,now)->(exp,sig)` / `verify(cid,exp,ct,sig,now)->bool`。
  - MAC = HMAC-SHA256(secret, `cid|exp|ct`) 的 hex。**cid+exp+ct 全入签名** → 不可篡改/挪用（换 cid/ct 即失效）、不可过期复用。
  - 验签用**常量时间比较**（`ct_eq`，等长 + 异或折叠无早退），防时序侧信道。
  - 无状态（密钥来自 env，多副本共享）→ 多副本签发/验签一致。
- **`GET /v1/asset/{cid}/bytes?exp&ct&sig`**：**不走 Bearer**。验签（HMAC + 未过期）即授权——拿到合法 URL 的前提是签发时过了 ACL（S3），字节端点只验签、不再查 ACL（守不变量 #3，presigned 标准语义）。验签通过 → `engine.fetch_inline_bytes(cid)`（S1）吐字节 + `Content-Type=ct`（ct 入签名 → 免 PG 再查 media_type）。未配签名器/无效/过期 → 403；无字节 → 404。仍受限流。
- env 装配：`FASTSEARCH_ASSET_SIGNING_KEY`（设即开启）+ `FASTSEARCH_ASSET_URL_TTL`（默认 300s）。
- 依赖：`hmac` 0.13 + `sha2` 0.11——**已在依赖树内**（transitive），加直接依赖零新增编译量。

## 怎么验证（本环境）

- 纯签名器 `asset_signer_sign_verify_roundtrip`：往返、未过期/过期边界、换 cid/ct/sig/密钥均拒。
- 端点（tower oneshot，无 source_pg 隔离 token 逻辑）：
  - 合法 token → **404**（验签过、但无 source_pg 无字节）——证明 token 已验、未误判 403。
  - 篡改 sig → **403**；过期 token → **403**；未配签名器 → **403**。
- 收口三绿：fmt + clippy --workspace -D warnings + test --workspace（30 套件，server 20 单测）。

## review 要点 / 诚实记账

- **安全**：MAC 覆盖 cid+exp+ct；常量时间比较；密钥泄露=可伪造（按 secret 管理、可轮换=旧 token 立即失效）；TTL 窗口内 ACL 变更有最长 TTL 滞后（短 TTL 控制，不引撤销列表——守无状态）。
- **未到 S3**：签发端点 `POST /v1/assets/resolve`（authed → resolve ACL → 对 InlineRef 用 AssetSigner 签 URL）尚未做 → 真用户暂无法取到 token URL（测试里直接构造）。S3 完成后"搜索→按 id 取 URL→`<img src>` 渲染"端到端可用。inline 真字节路（valid token + source_pg → 200 bytes）Docker PG 验证。
