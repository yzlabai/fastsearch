# devlog 2026-06-28 — CLI 重构为纯 REST 客户端

> 上游：[CLI 改为 REST 客户端设计](../plans/2026-06-28-CLI改为REST客户端设计.md)（含相似产品调研）。用户决策：CLI **完全改为纯客户端** + **"喂文件夹"保留**。spec 回写 [17-cli](../specs/17-cli.md)。

## 背景

CLI 原是**嵌入式引擎**（进程内 Tantivy 本地索引、本地搜索），是关键词-only（不算嵌入、向量不跨进程落盘）。问题诊断 + 用户讨论后定调：**CLI 应是 server 的客户端**，与 `clients/{python,ts}` 同模型。

**调研佐证**（设计 §10）：服务端检索/向量产品的 CLI 普遍是瘦 HTTP 客户端——Typesense（typesense-js 薄包装）、Qdrant（qcloud + qdrant-rest-cli 每端点一命令）、Algolia（云 API 客户端，profile+NDJSON+结构化 JSON）、Meilisearch（meilisearch-importer 批量上传）。嵌入式 CLI（tantivy-cli）属"嵌入式库"定位，与 fastsearch（服务端产品、PG 真源）不同。→ **CLI=瘦客户端是 norm**。

## 做什么

把 `fastsearch-cli` 从嵌引擎改为纯 REST 客户端：

- **删** `fastsearch-engine`/`fastsearch-text` 依赖（`cargo tree` 校验无）；**留** `core`（纯类型，做请求体/响应）+ `eval`（纯指标）；**加** `ureq`(json)。
- **`Client`**（ureq 瘦封装）：`--server`/`--key`（env `FASTSEARCH_SERVER`/`FASTSEARCH_KEY`，默认 localhost:8642）；`Authorization: Bearer`；非 2xx 带状态码报错、连接失败友好提示；`index` 走传输失败重试。
- **命令→端点**：`search`/`similar`→`/v1/search`·`/v1/similar`；`index`/`index-dir`/`ingest`→`/v1/index`；`eval`→ 语料入库 + 逐查询检索 + 本地指标。
- **喂文件夹保留**（用户要求）：`index-dir` 改为**客户端遍历+分块（`chunk_text` 纯函数）+ POST**，进度输出 + 逐文件 continue-on-error。经 server → **反得混合检索**（旧离线档只有关键词）。
- **解析留客户端**（`ingest` 的 docparse，`parse` feature）→ POST chunks，守"搜索热路径零 docparse" + CI 门禁。
- **collection 作用域**：核实 `collection` 在 text（term 翻译）+ vector（FieldSource）两端可过滤 → `--collection` 用 `Eq("collection",…)` filter 限定，**无需 server 改动**。
- **`--mode hybrid` 默认安全**：引擎 `want_vec` 要求 `query_vec.is_some()` → 无查询向量时自动退化关键词，不报错。

## 怎么验证

- **客户端单测**（无网，+7）：纯函数（parse_chunks/chunk_text/build_filter）+ **mock TcpListener**（search 解析 hits、index 取 indexed、index-dir 多文件上传、500 错误上浮）。mock 修了"响应前未排空请求体→RST→flaky"竞态（先读满 Content-Length 再响应）。
- **真二进制端到端**（起真 server，dev key）：`index-dir`(2 文件→5 chunk) → `search "毛利率"` 命中 `kb:report.md:1`、`--json` 全字段（bm25/heading_path/cursor…）；stdin `index` → `search "alpha"` 命中 `kb:manual:0`。ACL 经 `--key dev`、collection 经 filter 均生效。
- **收口三绿**：fmt + clippy（workspace + cli --features parse）-D warnings + test --workspace（30 套件）。`cargo tree -p fastsearch-cli` 无 engine/text。

## 取舍 / 诚实记账

- **CLI 不再离线**——用户已确认接受。喂文件夹改为联网上传 server；无 server 时 CLI 不可用。但换来**全套混合检索 + 真语义 + ACL**（旧嵌入式档只有关键词）。
- 连接配置仅 `--server`/`--key`+env；多 server profile（Algolia 式）、批量并发/ETA（Meilisearch 式）、对真 server 的 e2e 测试入 CI = 下一迭代。
- `eval` 会写入目标 server 的 golden 集合 → 文档标注用专用/临时集合。
- **作废** [三条线](../plans/2026-06-28-后续开发计划-三条线.md) 的"CLI 向量落盘"项（被本重构取代）。
