# devlog 2026-06-28 — MM6-signer S3：/v1/assets/resolve 签发端点（inline 档端到端就绪）

> 上游：[MM6-signer 设计](../plans/2026-06-28-MM6-signer-资产网关签名URL设计.md) 里程碑 S3；承接 [S1](2026-06-28-MM6-signer-S1.md)/[S2](2026-06-28-MM6-signer-S2.md)。

## 做什么（补上"签发"，inline 档端到端打通）

S1/S2 给了 InlineRef + 字节面 + token 验签；S3 加**签发端点**，让真用户能"按 id 取到可直接渲染的 URL"。

- **`POST /v1/assets/resolve {ids}`**（authed）：每 id → `resolve_citation(cid, acl)`（**ACL 服务端强制**）→ 按档：
  - `InlineRef` + 配了签名器 → `mint_inline_url` 签 `/v1/asset/{cid}/bytes?exp&ct&sig`（`type:"inline"` + `url` + `expires_s` + `media_type`）；
  - `SignedUrl`（Object）→ 返签名 URL（`type:"object"`）；
  - `DocRender` → 跳原文 JSON（`type:"doc_render"`）；
  - 越权/不存在 id → **直接省略**（不暴露存在性，守不变量 #3）。
- **`mint_inline_url`**：签名对**原始** cid/ct，URL 里 cid/ct **百分号编码**（`pct`，doc_id 可含空格/unicode → 路径/查询安全；字节端点 Path/Query 解码后验签一致）。`ct` 取 `media_type`（默认 octet-stream）。
- 前端用法：`POST /v1/assets/resolve {ids:[cid…]}`（带 Bearer）→ 拿 `url` → `<img src=url>`（免 Bearer，token 自带授权）。

## 怎么验证（本环境）

- `mint_inline_url_roundtrips_through_byte_endpoint`：mint 的 URL 直接喂 S2 字节端点 → 验签通过（非 403）→ 无 source_pg → 404。**S3 签发 ↔ S2 验签闭环**，且百分号编码可被端点正确解析。
- `assets_resolve_acl_not_bypassable`：灌 DocRegion 媒资图（acl team-a）→ resolve 以 team-a 得 1 条 doc_render、以 team-b 得空 assets（越权省略）。
- 收口三绿：fmt + clippy --workspace -D warnings + test --workspace（30 套件，server 22 单测）。

## 状态 / 诚实记账

- **inline 档"搜索→resolve→`<img src>`"端到端就绪**（本环境逻辑全绿）。inline **真字节路**（valid token + source_pg → 200 bytes）需 Docker PG 验证（同项目既有 gated 模式）。
- object 档真 presign = **S4 gated**（对象存储）；`/v1/assets/resolve` 的 OpenAPI 条目 = 小 follow-up（19-server spec 已文档化）。
- 安全：resolve 处强制 ACL（拿到 URL 的前提）；字节端点验签即授权（presigned）；越权 id 省略不暴露存在性。
