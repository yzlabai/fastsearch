## 1. MM2c-time — 时间区间列下推（已完成）

- **做什么**：pg 真源加 `time_start_ms/time_end_ms bigint`（由 `chunk.media.time` 派生落列）；B6 pgvector 直查把时间区间过滤从 `TRUE` 超集升级为**精确 SUPERSET 下推**（`col_kind` 纳入两列），与 modality/page 同构。
- **为什么**：音视频时间区间检索此前在 pg 直查路径只能 match-all + Rust 后过滤；下推让 PG ANN 在选择性时间过滤下仍 filter-aware（iterative scan）。后过滤 `PgVecRow` 仍取 `media.time` → 两端同源、守不变量 #5。
- **怎么验证**：收口三绿（fmt/clippy -D warnings/test --workspace）+ **Docker pgvector 真机**（DDL/16 参 INSERT/fetch 往返绿；`time_start_ms>=1000 AND time_end_ms<=5000` 仅命中区间内行）。新测试 `pgvector_sql_time_range_pushdown`、`chunkrow_media_roundtrip` 断言列值。
- **状态**：✅ 完成（commit `9099696`）。`media_bytes` 拆为 MM2c-bytes 随 MM6。
- **附带**：per-module spec 漂移回写（10-core/11-text/12-pg/16-embed），把已实现的多模态从代码回写进 spec。

## 2. MM5 — CDC `apply_upsert` modality 路由：无文本媒资不嵌（已完成 M0 部分）

- **做什么**：engine CDC 落地路径 `apply_upsert` 在 M0（仅文本嵌入器）下，**对无文本媒资 chunk（`text.trim()==""`，如无 caption 的图）跳过向量嵌入/写入**；幂等覆盖时若旧版本有向量则删除，避免残留。有文本（正文/caption/转录）照常嵌入。
- **为什么（真实问题）**：旧实现对所有 chunk 嵌 `chunk.text`。空串经 HashEmbedder 只剩前缀 token `passage` → **所有无文本媒资塌成同一向量**；真嵌入器给"空文档"向量，同样无意义 → 污染 ANN（一张图命中=全部无文本图同分命中）。plan M0 明确"退化为 caption 文本嵌入**或不嵌**"，这里取"不嵌"。无文本图仍在 BM25 + modality fast field（按模态可召回），视觉向量待 M1 图像嵌入路由。
- **怎么验证**：新单测 `mm5_textless_media_skips_vector`——经 `rebuild_from`（走 `apply_upsert`+HashEmbedder）灌入"有文本段 + 无文本图"，向量检索断言有文本段被召回、无文本图永不出现。收口三绿（fmt/clippy -D warnings/test --workspace，30 个测试套件全绿）。纯 Rust，无需 PG。
- **状态**：✅ M0 路由逻辑完成。**真 CDC 端到端音视频召回**（PG 写→复制→嵌入）仍需 `DATABASE_URL`，标 `待运行验证`；M1 图像向量路由 `gated`（需多模态模型）。
