# spec · fastsearch-pg

> 模块 #3，依赖：fastsearch-core。阶段 P1。上游：[产品设计 §2.3/§2.4](../plans/2026-06-24-产品设计文档.md)。
> 状态：**已落地 v1.5**（DDL/写路径/读取/pgvector 直查/媒资字节/B6 CDC 写穿，Docker pgvector 真机验证；见 §7）。

## 1. 目的与范围

Postgres 真源接入层：

- **Schema/DDL**：`fastsearch_chunks` 表 + 索引 + 逻辑复制 publication 的 DDL 生成（幂等 `IF NOT EXISTS`）。
- **Chunk ↔ 行映射**：core `Chunk` → SQL 参数（写入）；DB 行 → `Chunk`（读取/CDC 解码复用）。
- **写路径**：doc_id 级替换（DELETE WHERE collection,doc_id + 批量 INSERT）的语句生成 + 执行。
- **客户端**：`PgStore`（基于 tokio-postgres）连接、建表、写入、查询。

**不做**：CDC 解码（fastsearch-sync）、检索（fastsearch-text/vector）、pgvector ANN 查询的相关性逻辑（仅提供向量列读写）。

**可移植硬约束**：DDL 只用 pgvector 扩展（`CREATE EXTENSION IF NOT EXISTS vector`）+ 逻辑复制，**不依赖任何需 `shared_preload_libraries` 的原生扩展**（N1b）。

## 2. 数据结构与映射

- 表结构见产品设计 §2.3。列：`collection,doc_id,chunk_id,kind,text,page,bbox(jsonb),heading_path(text[]),section_id,char_len,modality(text NOT NULL DEFAULT 'text'),media(jsonb),tenant,acl(text[]),embedding(halfvec/可空),embed_model,updated_at`，主键 `(collection,doc_id,chunk_id)`。
  - **多模态（MM2a/MM2b，已实现）**：`modality` 由 `kind` 派生落列（过滤下推优化）；`media jsonb` 承载 `MediaRef`（不含 inline 字节）。**旧 `image_meta` 列已移除**，INSERT 14 参数，`media` 用 `$12::text::jsonb`（同 bbox 的 text→jsonb 强制转换）。
  - **时间列（MM2c-time，已实现，Docker 验证）**：`time_start_ms/time_end_ms bigint NULL`，由 `chunk.media.time` 派生落列（写侧反规范化，读路径 `time` 仍由 `media` 恢复，与后过滤同源）。`col_kind` 纳入这两列 → 时间区间过滤**精确 SUPERSET 下推**（`time_start_ms >= $n` 等），与 modality/page 同构；后过滤 `PgVecRow` 仍取 `media.time` → 两端同源、守不变量 #5。INSERT 16 参数。
  - **inline 字节列（MM2c-bytes，已实现，Docker 验证）**：`media_bytes bytea NULL`（PG 真源，承载 `AssetPointer::Inline` 的小裁图字节）。`Chunk.media_bytes` 字节通道 + `ChunkRow` 往返 + INSERT 17 参 + fetch SELECT 携带；**`PgStore::fetch_media_bytes(collection,doc_id,chunk_id)`** 供媒资网关按需直查（字节是真源、引擎派生层不持）。sync CDC 不搬字节（复制流只搬指针，§4.2）。字节生产者经 **`upsert_doc`**（库/外部 PG 写入路径）落字节；**CLI `index` 仅写本地索引、不写 PG**，故无 CLI 字节携带（CLI→PG 属独立命令特性，非本期）。设计见 [MM2c-bytes+MM6-inline 媒资服务设计](../plans/2026-06-27-MM2c-bytes与MM6-inline媒资服务设计.md)。
- 向量维度按集合配置（建表时定 `vector(N)`/`halfvec(N)`）。MVP 允许 embedding 可空（外部later 回填）。
- 映射函数（纯函数，可单测）：
  - `chunk_columns() -> &[&str]`：列名顺序（写入用）。
  - `insert_sql(collection_table) -> String`：参数化 INSERT。
  - `delete_doc_sql() -> String`：`DELETE ... WHERE collection=$1 AND doc_id=$2`。
  - `ddl(cfg) -> Vec<String>`：建扩展/表/索引/publication。

## 3. 公开接口

```rust
pub struct PgConfig { pub url: String, pub table: String, pub vector_dim: usize, pub vector_type: VectorType }
pub enum VectorType { Vector, HalfVec }

pub struct PgStore { /* tokio_postgres::Client */ }
impl PgStore {
    pub async fn connect(cfg: PgConfig) -> Result<Self>;
    pub async fn ensure_schema(&self) -> Result<()>;                 // 幂等 DDL
    pub async fn upsert_doc(&self, collection: &str, doc_id: &str, chunks: &[Chunk]) -> Result<u64>; // 替换
    pub async fn delete_doc(&self, collection: &str, doc_id: &str) -> Result<u64>;
    pub async fn fetch_doc(&self, collection: &str, doc_id: &str) -> Result<Vec<Chunk>>;
}
```

- 纯 SQL/映射函数独立于 client，便于单测（无需 PG）。
- 错误 `PgError`（含 tokio_postgres 错误、映射错误）。

## 4. 行为规约

- **doc_id 级替换**：`upsert_doc` 在一个事务里先 `DELETE WHERE collection,doc_id` 再批量 INSERT，保证原子（CDC 侧看到 delete+insert）。
- **幂等 DDL**：`ensure_schema` 可重复调用。
- **REPLICA IDENTITY**：表用默认（PK）即可捕获删除键；publication 覆盖该表。
- **Publication 列清单（B6 写穿，断 CDC 反馈环）**：publication 用**列清单**只发布源真源列（`COLUMNS`），**排除引擎派生/记账列** `embedding`/`embed_model`/`updated_at`——`set_embedding` 写穿这三列**不产生复制事件**，从根断开"写穿→复制→再嵌入→再写穿"环。需 **PG 15+**（列清单为核心特性、非扩展，不破不变量 #1）。DDL 幂等且**自愈不抢占**：无→`CREATE`（带 `EXCEPTION WHEN duplicate_object/unique_violation` 防并发首建竞态）；本表 legacy 全列发布（`prattrs IS NULL`）→ `ALTER SET TABLE` 收敛；属别表的同名 publication 不动。
- **写穿幂等守卫**：`set_embedding` 的 `WHERE ... AND (embedding IS DISTINCT FROM $1 OR embed_model IS DISTINCT FROM $5)`——值未变 → 0 行更新 → 不空转复制（与列清单互为防线）；`clear_embedding` 仅在 `embedding` 非空时更新。
- **健壮**：连接/SQL 错误上报不 panic；bbox/heading_path/acl 的 (de)serialize 失败显式报错。

## 6. 测试用例

**单元（无需 PG，必跑）**：
1. `ddl()` 含 `CREATE EXTENSION IF NOT EXISTS vector`、建表含全部列与主键、publication 语句、`halfvec(384)` 维度正确。
2. `insert_sql()`/`delete_doc_sql()` 形态正确（占位符数、WHERE 条件）。
3. Chunk → 参数行：bbox→jsonb 字符串、heading_path→text[]、acl→text[]、kind→snake_case 字符串、可空 tenant/embedding 处理。
4. 行 → Chunk：往返一致（含中文、含 ':' 的 doc_id、空 heading_path）。

**集成（需 PG，env-gated，`DATABASE_URL` 未设则跳过/ignore）**：
5. ensure_schema → upsert_doc(3 chunk) → fetch_doc 得回 3 个、字段一致。
6. upsert_doc 替换：同 doc 再 upsert 2 个，fetch 得 2 个（旧的被删）。
7. delete_doc 后 fetch 空。
