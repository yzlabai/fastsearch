> 状态：**v1（含基准数据）**｜日期：2026-06-26｜上游：[迭代循环看板](../plans/2026-06-25-迭代循环.md) C2。
> 基准来自 [`tests/bench_capacity.rs`](../../crates/fastsearch-engine/tests/bench_capacity.rs)，
> **开发笔记本、release、N=10k、dim=96** 实测——为**指示性基线**，生产数字以目标硬件复跑为准。

## 1. 基准数据（指示性）

| 档 | 灌入吞吐 | keyword p50/p95 | vector p50/p95 | recall@10 |
|---|---|---|---|---|
| **Brute（暴力，默认）** | ~397k chunks/s（仅文本+内存向量） | 0.28 / 0.31 ms | 1.16 / 1.36 ms | 1.0（精确） |
| **HNSW（u8 量化）** | ~4.3k chunks/s（建图慢） | — | 1.13 / 1.24 ms | 0.984 |

复跑：`cargo test -p fastsearch-engine --test bench_capacity --release -- --ignored --nocapture`。

**读法**：
- **keyword 亚毫秒**（Tantivy mmap）。
- **向量**：N=10k 时 brute≈HNSW（暴力 1万条已很快）；**HNSW 的亚线性优势在更大 N（10^5-10^6+）才显现**，
  代价是建图吞吐低约两个数量级 + 近似（recall<1）+ 非确定。
- 选档准则：**≤~10^5 向量用 Brute**（精确、确定、建快）；**10^5+ 用 HNSW**（亚线性、可接受近似）；
  **不想吃引擎内存 / 已重用 PG → pgvector 直查档**（ANN 在 PG 跑，B6）。

## 2. 内存容量估算

引擎为**无状态派生索引**（可重建），单副本常驻内存约：

- **向量**：
  - Brute / HNSW-rerank 全精度：`N × dim × 4 B`（f32）。例 1M×384 ≈ **1.5 GB**。
  - HNSW 额外 u8 量化图：`N × dim × 1 B` + 图连接（`≈ M×8B×N`，M=16 → ~128B/向量）。
    例 1M×384：u8 ~0.4GB + f32 旁挂 1.5GB + 链接 ~0.13GB ≈ **2.0 GB**。
- **文本（Tantivy）**：与正文体量成正比，mmap 由 OS 分页（非全驻留）。粗估 ≈ 语料字节的 0.5–1×。
- **元数据/ACL**：每 chunk 数百字节量级。

**容量公式（单副本，brute）**：`RAM ≈ 向量(N×dim×4) + 文本 mmap 驻留 + 开销`。
按目标 N/dim/语料代入；超内存即水平分片（多副本各承载子集）或换 pgvector 直查（向量移出引擎）。

## 3. SLO 目标（建议，按基线设定）

| 指标 | 建议 SLO | 依据 |
|---|---|---|
| keyword 检索 p95 | < 10 ms | 基线 0.31ms @10k，留足放大余量 |
| hybrid 检索 p95 | < 50 ms | 含向量召回 + 融合 + 可选 rerank/高亮 |
| 可用性 | 99.9% | 无状态多副本 + PG HA（见 §4） |
| CDC 同步滞后 | < 5 s（p95） | 后台循环拍间隔可调（`FASTSEARCH_CDC_INTERVAL_MS`） |
| 相关性 nDCG | 不回归（CI 门禁） | eval golden 回归门禁守 |

> 注：consume 期间持引擎锁（与检索串行），高写入下检索延迟会被 CDC 拍影响——低延迟化（读写分离/
> 索引并发）见引擎并发优化（下一迭代）。

## 4. HA（高可用）

**引擎层无单点**：
- **无状态 + 派生可重建**：每副本各自从 PG 复制流 / 快照（`bootstrap_snapshot`）重建派生索引；
  崩溃恢复 = 重放 CDC（checkpoint 续传，幂等）或从快照重建（`Engine::open`）。
- **水平扩展**：多副本无共享状态，前置负载均衡即可（见 [deploy/cloudnativepg.yaml](../../deploy/cloudnativepg.yaml) replicas=2）。
- **ACL 不可绕过**在每副本服务端注入，扩容不削弱安全。

**真源（PG）层 HA**：CloudNativePG 3 副本（1 主 2 从），仅 pgvector + 逻辑复制，无原生扩展
（托管 PG 可移植，见 [许可/部署](2026-06-26-词典与第三方许可审.md) 与 deploy/）。

**恢复目标**：
- RTO（引擎副本）：≈ 重建派生索引时间（bootstrap：基线灌入 ~397k chunks/s → 1M 约数秒~分钟级）。
- RPO：≤ CDC 拍间隔（默认秒级），PG 侧由其复制保障。
