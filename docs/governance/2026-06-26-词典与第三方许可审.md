# C1 · 中文词典与第三方许可审（分发前）

> 状态：**已审**｜日期：2026-06-26｜上游：[迭代循环看板](../plans/2026-06-25-迭代循环.md) C1。
> 结论：**当前依赖无 CC-BY-SA 等 share-alike 词典义务**；分发时按 MIT 保留 jieba-rs 版权与许可文本即可。

## 1. 背景与关注点

看板 C1 originally 担心"lindera/jieba 中文字典许可（CC-BY-SA）"——CC-BY-SA 的 **share-alike**
会要求衍生分发同样以 CC-BY-SA 开放，可能与本项目 Apache-2.0 冲突。**审下来这个担心不适用于
fastsearch 的实际依赖**（见下）。

## 2. 实际用了什么（代码是真源）

- 唯一分词依赖：**`jieba-rs` 0.10.1**（`crates/fastsearch-text` → `TokenizerKind::Jieba`）。
- jieba-rs 默认启用 `default-dict` feature，**内嵌词典** `src/data/dict.txt`（~5MB）+ `idf.txt`，
  随二进制分发。
- 许可：jieba-rs `Cargo.toml` `license = "MIT"`；README 明确 "released under the MIT license"；
  `dict.txt` 内无独立版权/许可声明，**由该 crate 的 MIT 整体覆盖**（源自 MIT 的
  fxsjy/jieba 项目）。
- **未使用 `lindera`**（其常配的 CC-CEDICT / 部分中日韩词典才涉及 CC-BY-SA / 各自许可）——
  `grep -r lindera crates/*/Cargo.toml` 为空。CC-BY-SA 的担心来自 lindera 生态，与本项目无关。
- `TokenizerKind::Default`（空白/标点切分）**不含任何词典**，无许可问题。

## 3. 许可相容性

| 组件 | 许可 | 与 Apache-2.0 分发 |
|---|---|---|
| fastsearch 本体 | Apache-2.0 | — |
| jieba-rs 0.10.1（含内嵌 dict.txt/idf.txt） | MIT | ✅ 相容（MIT 可并入 Apache-2.0 分发） |

MIT 与 Apache-2.0 单向相容：可在 Apache-2.0 作品中分发 MIT 组件，**义务仅为保留 MIT 的版权声明
+ 许可全文**（无 share-alike、无传染）。

## 4. 分发合规清单（发布前做）

1. **保留归属**：随二进制/发行物附带第三方许可清单，至少含 jieba-rs 的 MIT 全文 +
   版权行（`Copyright (c) 2018-2019 messense; Copyright (c) 2019 Paul Meng`）。
2. **生成方式**（建议）：用 `cargo about generate` 或 `cargo bundle-licenses` 产出
   `THIRD-PARTY-NOTICES.md`，纳入发布产物（容器镜像 `/usr/share/...` 或发行 tarball 根目录）。
3. **CI 守门**（✅ 已落地 2026-06-30，cargo-deny 0.19.9 本机实跑验证 `licenses ok, sources ok`）：
   根 [`deny.toml`](../../deny.toml) + CI `licenses` job 跑 `cargo deny check licenses sources`
   （硬门禁），allow-list = Apache-2.0 / MIT / BSD-2-Clause / BSD-3-Clause / ISC / Unicode-3.0 /
   Zlib / MPL-2.0 / CDLA-Permissive-2.0（均宽松、无 share-alike，且**实测全部被命中、无冗余**），
   拒绝意外引入 GPL/CC-BY-SA 等 copyleft/share-alike 依赖；`bans`（多版本/通配）+ `advisories`
   （RustSec）信息性不阻塞（内部 path 依赖会被判通配=误报，新 CVE 不应让无关 PR 变红）。
   **实测审计**（`cargo metadata` 全树）：无任何**唯一** copyleft/share-alike 许可，其余 SPDX OR
   表达式均解析到宽松项；唯一「非 MIT/Apache 的唯一许可」为 ONNX 生态 CDLA-Permissive-2.0
   （宽松、已入 allow-list）。许可清单经 [`about.toml`](../../about.toml) + `cargo about generate`
   产出 `THIRD-PARTY-NOTICES.md`（CI artifact，随发布物存档）。
4. **换词典需重审**：若未来引入 lindera 或自定义 CC-CEDICT 词典，**必须重新评估**（CC-BY-SA
   share-alike 会改变分发义务）。本文件即为该重审的基线——`deny.toml` 把它机器化：share-alike
   许可一旦入树即让 CI 变红。

## 5. 结论

- **现状放行**：当前依赖（jieba-rs MIT）无 share-alike 义务，Apache-2.0 分发合规，只需附 MIT 归属。
- **C2（容量/SLO）** 与本许可审无关，单列。
- 复核动作（`cargo about`/`cargo deny`）✅ 已落地为 CI `licenses` job（§4.3），机器化守门 +
  清单产出。
