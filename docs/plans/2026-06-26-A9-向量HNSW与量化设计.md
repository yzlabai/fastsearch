## 1. 为什么 / 何时

当前 `MemVectorIndex` 是**暴力精确**余弦检索（打分前真预过滤），在 ~10^5 向量内延迟可接受、召回=1.0。A9 解决**规模**：到 10^6+ 向量时暴力线性扫描过慢。HNSW 是**近似最近邻（ANN）**图索引，亚线性查询，代价是召回 < 1.0（可调）。

**定位**：性能基础件（不是修正项）。**默认仍走暴力**；HNSW 是 opt-in 的大规模档位。

## 2. 依赖与可移植（守不变量 #1）

- 选 `hnsw_rs`（**确认纯 Rust、无 C/原生依赖**——施工前 `cargo tree` 核验；若引入 C 依赖则改用纯 Rust 备选或自实现 HNSW 核心）。
- 不变量 #1 是关于 **PG 侧**不装原生扩展；引擎侧 Rust 依赖只要纯 Rust 即不影响单二进制分发。仍倾向纯 Rust，避免交叉编译麻烦。

## 3. 架构：新后端，trait 边界接入

`vector` crate 已有 `VectorBackend` trait（`upsert` / `search`）。新增 `HnswVectorIndex` 实现它，与 `MemVectorIndex` 并存：

```
VectorBackend (trait)
  ├── MemVectorIndex   暴力精确（默认；小规模/CI/fallback）
  └── HnswVectorIndex  HNSW 图 + int8 量化向量 + 旁挂全精度 f32（大规模 opt-in）
```

- `engine` 持 `Box<dyn VectorBackend>` 或枚举二选一（按配置）。**对 engine 排序管线零改动**——trait 后面换实现。
- 选择入口：`VectorBackendKind { Brute, Hnsw{ m, ef_construction, ef_search } }`，由 engine 配置/env 决定。

## 4. filter-aware：HNSW 的预过滤如何不破不变量 #5

不变量 #5（SUPERSET 后端查询 + `Filter::eval`/`AclFilter::visible` 精确后过滤）本是讲**过滤翻译不漏召回**。HNSW 叠加一层**ANN 近似**，要分清两件事：

| 维度 | 保证 | 手段 |
|---|---|---|
| **精度 / ACL** | **精确、不越权**（硬约束，不可近似） | ANN 取回候选后，仍用 `core::Filter::eval` + `AclFilter::visible` **精确后过滤**（同暴力路） |
| **召回** | **近似**（ANN 固有，可调、需诚实标注） | over-fetch：`ef_search` 与取回候选数 `≫ top_k`；过滤掉的候选不回填则该页变少 |

**关键风险**：强过滤（命中集很小）下，ANN 取回的 top-N 候选可能**几乎都被过滤掉** → 召回骤降。缓解（按复杂度递增，先做①②）：
1. **over-fetch 系数**：取回 `top_k × over_fetch`（如 ×10）候选再后过滤。
2. **小集合回退暴力**：若过滤后候选 < 阈值且总量不大，回退 `MemVectorIndex` 暴力（精确）。施工时 engine 侧可保留全精度向量，过滤性强时直接暴力。
3. **（下一迭代）filtered traversal**：把过滤谓词下推进图遍历（跳过不满足节点），召回更稳但实现复杂、与 `hnsw_rs` API 耦合。

> 诚实记账：A9 落地后召回**不再是 1.0**。必须在 spec 标注"HNSW 档为近似召回，recall@k≈X（见门禁）"，不得说成精确。

## 5. 量化 + 全精度重排

- **图向量 int8 标量量化**：每维线性量化到 i8（per-vector 或 global min/max）。内存 ↓4×、距离计算更快。RaBitQ（二值旋转量化）召回/压缩更优但复杂 → **下一迭代**。
- **旁挂全精度 f32**：保留原始 f32 向量（或从 PG 取）。流程：HNSW 在 int8 上粗排取 `top_k × rerank_factor` 候选 → 用 f32 **精确重算余弦**重排 → 截 top_k。即"量化粗筛 + 全精度精排"，抵消量化误差。

## 6. 持久化（接 engine `persist`/`open`）

- `hnsw_rs` 支持 graph dump/load；`HnswVectorIndex::save/load` 序列化图 + int8 向量 + f32 旁挂 + `VecMeta`。
- 接入 `Engine::persist`（落 `<data>/vector_hnsw.*`）与 `Engine::open`（加载）。`Checkpoint.vector_dim` 已有；增记后端类型，换档需重建（派生可重建，走 `rebuild_from`）。

## 7. 确定性（守不变量 #4）

HNSW 建图依赖随机层分配 + 插入序 → 不同建图可能得不同近似图。调和：
- **固定 RNG 种子**建图 → 给定同一插入序得同一图。
- 不变量 #4 措辞是"同输入 + **同稳定索引** → 同结果"：**给定一个已建好的图快照**，检索遍历确定 → 结果确定；最终融合仍按 `GlobalId` tie-break。
- 诚实标注：**重建图**（不同插入序/并发）可能得到不同近似图（不同近似召回），但**每个稳定快照内确定**。这是 ANN 的固有性质，明示之。

## 8. 测试计划（本环境，纯 Rust）

1. **召回门禁**：合成 N=2000 随机向量，HNSW vs `MemVectorIndex` 暴力 ground-truth，断言 `recall@10 ≥ 0.95`（参数调到达标）。纳入回归（掉点失败）。
2. **filter-aware 正确性**：带 filter/ACL 检索，断言结果 ⊆ 暴力精确结果（**精度/ACL 不放松**），且无越权泄漏。
3. **量化精度**：int8 + 全精度重排 vs 纯 f32 暴力，top-k 重合率达阈值。
4. **持久化往返**：save→load 后检索结果一致。
5. **确定性**：固定种子两次建图 + 检索，结果一致。
6. **小集合回退**：强过滤下回退暴力，召回=1.0。

## 10. 风险与回退

- `hnsw_rs` API/质量不达预期 → 回退自实现最小 HNSW 或保持暴力（A9 标 `下一迭代`，不强上）。
- 强过滤召回坑 → 小集合回退暴力（§4②）兜底。
- 默认档不变（暴力），HNSW 全程 opt-in → 上线零风险，按需开。
