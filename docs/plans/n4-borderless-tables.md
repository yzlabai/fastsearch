# Plan · N4 无框表格检测（数据驱动的最高结构优先）

> 上层 [next-iteration.md](next-iteration.md) N4。N1 同台数据坐实：含表文档表格检出召回 **1/6**、TEDS≈0——学术 PDF 多为**无框**（booktabs）表格，是与 Docling 的最大结构差距。
>
> **反馈回路**：用 `scripts/eval/compare_docling.py` 度量改动是否真把 TEDS/召回拉起来（数据驱动）。

## 思路：靠对齐而非 ruling line

无框表格没有矢量线，但**单元格按列对齐**。检测信号：连续多行的文本，其"单元格"（行内被大间隙分开的片段）对齐到一致的列 x 位置。

```
文本 chunk → 行(共基线) → 行内按大间隙切 cell → 连续行共享列结构 → 无框表格
```

落点：`core::table` 加 `detect_borderless_tables(chunks) -> Vec<Table>`；interpreter 在**有框检测之后**、对**未落入有框表**的 chunk 再跑无框检测。

## 算法（保守）

1. 行重建：chunk 按基线分行，行内 chunk 按 x0 排序。
2. 行内切 cell：相邻 chunk 间隙 > `CELL_GAP`(≈1.5em·字号) 即列边界；否则同 cell（多词单元格内 0.25em 词距不切）。
3. 候选行：cell 数 ≥2 的行。
4. 区域生长：从一候选行起向下扩，只要下一行**垂直连续**（间隙≈行距）且其 cell 左缘**对齐**到区域已有列（容差内）。
5. 成表门控（防误判，关键）：≥3 行 且 ≥2 列 且 列在 ≥70% 行上被填充 且 区域连续。
6. 建表：列 = 聚类后的 x 左缘；每行 cell 归到最近列 → 网格；cell 文本复用 `reconstruct_lines`。

## 防误判（硬门控）

- ≥3 行、≥2 列、列稳定（多数行填多数列）。
- 不与有框表 bbox 重叠。
- 单一大间隙（标题+值）不算表。
- 多栏正文已被 XY-cut 分列处理（在单列内检测），降低"把双栏正文当表"的风险。

## 验收

- **量化**：`compare_docling.py` 表格检出召回从 1/6 上升、含表子集 TEDS 上升（born-digital LTR）。
- **零回归**：lorem/1901 正文不被误判为表；bialetti 有框表仍对；三件套文本不退化；确定性。
- `core::table` 无框检测单测（合成对齐网格→检出；散排正文→不检出）；clippy 零 warning。
- **不做**：合并单元格 span、嵌套表、跨页表合并（留后续）。标题分级（MHS）另立增量。

## 风险

- 误判是最大风险——宁可**漏检**（保守）也不把正文/图注当表。先严门控，用真实样例（1901/2408 正文、bialetti）回归确认零误判，再用 Docling 学术表验证召回。
