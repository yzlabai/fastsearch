## 1. 为什么 / 何时

当前 RaBitQ/TurboQuant 的数据无关旋转用**物化 d×d 高斯矩阵 + Gram-Schmidt**
（`binary::Rotation`）：**存储 O(d²)、建矩阵 O(d³)、apply O(d²)/向量**（标量，`unsafe` 禁 → 无 SIMD）。
这既是 DoS 面（[决策记录](../governance/2026-07-21-向量旋转维度上限与DoS.md)：小快照声明巨维 → 巨额分配，
已用 `MAX_ROTATION_DIM=8192` 临时封顶），也是**高维摄取的真实瓶颈**。

**FHT 结构化旋转**（随机化 Walsh-Hadamard / Fast Johnson-Lindenstrauss）把旋转变成
**存储 O(d)、apply O(d·log d)、零建矩阵成本**：

| d | 物化 apply/向量 | FHT apply/向量 | 物化存储 | FHT 存储 |
|---|---|---|---|---|
| 1536 | 2.36M | **0.023M（~100×↓）** | 9.4 MB | ~8 KB |
| 3072 | 9.44M | **0.049M（~190×↓）** | 38 MB | ~16 KB |
| 4096 | 16.8M | **0.049M** | 67 MB | ~16 KB |

**收益**：① 消掉 DoS 分配（无 d×d 矩阵，`MAX_ROTATION_DIM` 对 FHT 档可大幅放宽/取消）；
② 高维摄取快 ~100–200×；③ 纯 Rust O(d log d) 无 BLAS——**turbovec 自身用物化 d×d（靠 BLAS 快），
fastsearch 上 FHT 反而在"纯 Rust 无 BLAS"约束下超过它**。

**定位**：旋转的**实现替换**（同为数据无关、确定、正交），对上层量化/估计器**语义不变**（见 §3）。

## 2. 变换定义

对输入向量 `x`（长度 d），**零填充到 `D = next_pow2(d)`**，做 `ROUNDS` 轮
`[随机符号翻转 Dₛ → 归一化 Walsh-Hadamard Hₙ]`：

```
R(x) = Hₙ·D_s3·Hₙ·D_s2·Hₙ·D_s1 · pad_D(x)     （ROUNDS=3）
```

- `pad_D`：末尾补零到 D。
- `Dₛ`：对角 ±1（每维独立，**固定种子** xorshift 生成，各轮独立）——正交。
- `Hₙ = H/√D`：D×D Walsh-Hadamard 矩阵归一化——正交（`H·Hᵀ=D·I`）。**原地蝶形迭代
  O(D·log D)、纯 Rust 无 unsafe、无需物化 H**。
- 输出长度 **D**（非 d）。多轮把能量摊得更均匀（单轮已够摊，多轮更接近各向同性）。

## 3. 正确性性质（为何上层语义不变）

- **正交 → 保内积**：`R` 是正交阵之积 → 正交。零填充不改内积（`⟨pad(q),pad(v)⟩=⟨q,v⟩`），
  正交变换保内积 → **`⟨R(q),R(v)⟩ = ⟨q,v⟩`**（至 f32 舍入）。故量化估计器 `⟨q_rot,x̂_rot⟩·corr`
  仍估 `⟨q,v⟩`、二值符号码/精排语义**不变**（精排仍可用原向量或等价）。
- **能量摊匀（各向异性增益）**：任一输入方向经 HD 后各坐标 ≈ 均匀幅度 → 符号/低比特码更有信息、
  旋转后坐标 ≈ N(0,1/D)，**Lloyd-Max codebook 数学照旧**（`quant` 的标准化 `s[i]=u[i]·√(len)`
  用 `len=D` 自动成立，因 apply 输出即 D 长）。
- **确定性（守 #4）**：`Dₛ` 由固定种子生成、Hadamard 确定 → 同 `(d, seed)` 同变换；同分 `GlobalId`
  tie-break 不变。
- **无新依赖 / 无 unsafe**：蝶形是安全 Rust；符号用现成 xorshift。

## 4. 代价权衡（诚实记账）

- **填充膨胀码**：非 2 幂维 D>d → 码宽按 D 算。1536→2048、3072→4096 = **+33% 码字节**（2 幂维 0%）。
  换来旋转存储 O(d²)→O(d)、apply ~100–200×↓、无建矩阵——**净大赢**（码 +33% vs apply 省 ~190×）。
- **多轮 vs 单轮**：ROUNDS=3 给稳健各向同性（O(D log D) 常数 ×3，仍远小于物化）。可调常量。
- **非精确等距**：结构化旋转比满随机旋转略欠各向同性（有限轮）→ 各向异性极端集召回可能微逊满旋转；
  以召回门禁守（§8），实测对比物化档。

## 5. 集成方案

> **落地回写（代码是真源）**：

- ✅ **Step 1 原语** `fht::StructuredRotation`：`new(dim, seed)`（存 `D + ROUNDS×D 符号`）+
  `apply(&[f32]) -> Vec<f32>`（长 D）+ `out_dim()`。纯 Rust 无 unsafe/无依赖。（未存原始 d——
  上层 turbo 自持逻辑维，原语只需 D。）
- ✅ **Step 2 接 turbo**：`TurboVectorIndex` 用 `StructuredRotation` 替物化 `Rotation`；
  `packed_len`/`query_lut`/`score` 均按 **D=`rotation.out_dim()`**；快照存**原始 d**、D 派生；
  `search` 用 `q_rot.len()` 作打分维。**决策：无条件替换**（非 opt-in flag）——turbo 新档无生产数据，
  FHT 严格更优（快/省）**且实测召回不劣**（`recall_vs_exact_fht` d=1000→D=1024：4-bit exact@10
  **0.885** vs 物化 ~0.87），故不引 `fht:true` 档/plumbing。**快照 v1→v2**（码空间从 d 变 D，
  v1 不兼容 → load 拒之，无生产 v1 数据）。**物化 `Rotation` 保留给 `BruteBinaryRotated`**（未动）。
- **Step 3（下一迭代，可选）**：`BruteBinaryRotated` 亦切 FHT；退役物化 `Rotation` 或留 fallback；
  FHT 档放宽 `MAX_DIM`（无 d×d 分配，8192 sanity 界可大幅提高）。

## 6. 测试计划（纯 Rust、本环境）

Step 1（原语）：
1. **保内积**：随机 q,v（多维含非 2 幂如 768/1536），`⟨R(q),R(v)⟩ ≈ ⟨q,v⟩`（f32 tol）——**正交核心**。
2. **保范数**：`‖R(x)‖ ≈ ‖x‖`。
3. **能量摊匀**：尖峰输入 `[1,0,…]` → 输出各坐标幅度方差小（≈均匀 `1/√D`）。
4. **确定性**：同 `(d,seed)` 两次 apply 逐位一致。
5. **无巨分配**：d=4096 apply 正常（物化档此维要 67MB+O(d³)，FHT 秒级、KB 级）。
6. **蝶形正确**：小维 Hadamard 对拍已知 H₂/H₄ 结果。

Step 2（接 turbo）：✅ **召回门禁** `recall_vs_exact_fht`（d=1000→D=1024，4-bit exact@10 **0.885**
≥0.82 门禁，与物化档 ~0.87 同量级）；既有 turbo 8+ 测（持久化往返/确定/维度校验/排序/预过滤/ACL/
毒丸/超维）在 FHT + v2 下仍全绿（回归）。

## 7. 施工顺序（每步收口三绿 + 回写）

1. ✅ **`StructuredRotation` 原语**：pad + ROUNDS×(符号翻转 + 原地 WHT) + 归一化；+7 测 §6.1–6.6。
2. ✅ **接 turbo**：D 记账（`out_dim`）+ 快照 v2 + **无条件替换**（recall parity 实测证成立，非 opt-in）+
   召回门禁；`mod fht` 转无条件。
3. ✅ **接 `BruteBinaryRotated` + 退役物化 `Rotation`**：`MemVectorIndex` 旋转切 FHT（code/l1 转 D 空间、
   精排仍用原 d 向量）；**删除 `binary::Rotation`/`next_unit`**（FHT 成唯一旋转实现）；`MAX_ROTATION_DIM`
   移到 `fht::MAX_DIM`（含义从"d×d DoS 闸"变"sanity/溢出界"，值 8192 不变）；`ensure_rotation`/
   `set_rabitq_rotation` revert 非 Result（FHT 无巨分配）。**实测**：各向异性 recall FHT **0.925** vs 非旋转
   0.775（+15pt，>+5pt 门禁；略低于物化 0.97——3 轮 HD 各向同性稍欠，实测 ROUNDS=3 turbo 最优）。
   **MAX_DIM 未放宽**（保 8192）：真实维 ≤4096 已覆盖，放宽是 speculative（避 code smell）。

## 8. 风险与回退

- FHT 各向同性略欠 → 召回门禁守：**实测 0.885 ≥ 物化 ~0.87，不劣**（3 轮 HD 足够）。
- 填充膨胀码 → 已记账；2 幂维无膨胀，非 2 幂 +33% 可接受（换 apply ~100–200×↓）。
- turbo 无条件用 FHT（v2）；`BruteBinaryRotated` 仍物化档不变 → 默认档（Brute/BruteBinary）零影响。
  若未来 FHT 在某数据召回不达标，可回退：turbo v3 加 `fht` flag + 保留物化路径（当前实测无需）。
