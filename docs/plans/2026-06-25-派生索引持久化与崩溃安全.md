# 派生索引持久化 + 崩溃安全的 CDC 检查点

> 状态：**计划（待 review → 实施）**｜日期：2026-06-25｜跨模块：`vector` / `engine` / `sync`。
> 上游：[产品设计 §崩溃恢复/生命周期](2026-06-24-产品设计文档.md)（F47–F49）、[13-sync spec §4 快照+增量](../specs/13-sync.md)、[迭代循环看板](2026-06-25-迭代循环.md)。
> 不变量：PG 是真源、引擎索引派生可重建（CLAUDE.md §2）；幂等 + LSN 水位（§4）。

## 1. 目的与范围

让派生索引能**跨重启/崩溃存活**，并修复当前 CDC 消费的一个**真正的丢数据风险**。

**要做：**
1. **向量索引持久化**：`MemVectorIndex` 落盘（原子写）+ 启动加载。文本索引（Tantivy/mmap）已落盘，本计划补齐向量这半边。
2. **崩溃安全检查点**：把 CDC 消费从 `get`（拉即推进 slot）改为 **`peek`（看但不推进）→ 应用 → 持久化索引+检查点 → 才 `pg_replication_slot_advance`**；持久化 `applied_lsn`。
3. **重启续传**：从持久化检查点恢复，向量**不重新嵌入**（嵌入是最贵的一步）。
4. 与已设计的**初始快照**（13-sync §4）、未来**流式 CDC** 的衔接（同一套"先落盘后推进"纪律）。

**不做（本轮）：** 蓝绿双版本/原子指针切换（F47）、索引格式在线迁移（F49，仅留 `schema_version` 字段占位）、流式 `START_REPLICATION`（仍用 peek 轮询；纪律相同）、HNSW。

## 2. 当前缺口（为什么必须做）

- 向量索引 `MemVectorIndex` 纯内存 → 重启即丢 → 混合检索**静默退化为 keyword**，直到全量重建（且重建要**逐 chunk 重嵌入**，压垮嵌入服务）。
- `replication::pull_changes` 用 `pg_logical_slot_get_binary_changes`（消费即推进 slot）。**拉取后、索引落盘前崩溃 → 这批变更 slot 已认为消费完、不再重发，索引里却没有 → 永久丢数据。** 13-sync 现有"崩溃从 slot 续传"的表述**高估了安全性**，须更正。
- `Applier.applied_lsn` 只在内存，无持久化检查点。

## 3. 设计

### 3.1 落盘布局（沿用 engine data 目录）

```
<data>/
  text/                # Tantivy（已落盘，mmap）
  vector.bin           # 向量索引快照（bincode/JSON；原子写）
  checkpoint.json      # { schema_version, applied_lsn, vector_dim, embed_model }
```

- **原子写**：写 `*.tmp` → `fsync` → `rename`（rename 原子），避免写一半崩坏。
- `MemVectorIndex` 需要可序列化：给 `VecMeta`、`Entry` 加 `serde::{Serialize,Deserialize}`（`GlobalId` 已具备）；或定义独立快照 DTO。提供 `save(&Path)` / `load(&Path)`。
- `checkpoint.json` 记 `applied_lsn`（u64）+ `vector_dim` + `embed_model`（用于检测"换了模型/维度需重建"）+ `schema_version`（F49 占位）。

### 3.2 一致性与崩溃安全模型（核心）

消费一批变更的**严格顺序**（`engine::consume_once` 编排）：

```
1. peek_changes(slot)              # 不推进 slot
2. applier.apply_batch(engine)     # 幂等应用到内存 text+vector（apply_upsert 含嵌入）
3. engine.persist(data)            # text.commit() → vector.save(原子) → checkpoint.json(原子)
4. advance_slot(slot, last_lsn)    # 持久化成功后，才推进 slot
```

**铁律：slot 的推进（步 4）必须发生在索引+检查点持久化（步 3）之后。** 由此：

| 崩溃点 | 重启后行为 | 结论 |
|---|---|---|
| 步 3 前 | checkpoint 仍是旧值；slot 未推进 → 重 peek 拿到同批 → 重放（幂等） | 不丢、可能重嵌少量尾部（可接受） |
| 步 3、4 之间 | checkpoint 已新；slot 未推进 → 重 peek 拿到同批 → `Applier` 跳过 `lsn ≤ applied_lsn` | 不丢、不重复（幂等去重） |
| 步 4 后 | checkpoint 与 slot 一致 | 正常续传 |

→ **至少一次投递 + 幂等应用 = exactly-once 效果**，且无论何时崩溃都不丢、不重。

启动流程：
```
load checkpoint.json → applied_lsn
load vector.bin（dim/model 不符 → 警告并触发重建路径）
text 索引由 Tantivy 自动 mmap 打开
Applier::new(applied_lsn) → 之后 consume_once 从该水位续传
```

### 3.3 API 变更

- `vector`：`MemVectorIndex::save(&Path)` / `load(&Path) -> Result<Self>`；`VecMeta`/`Entry` 加 serde。
- `sync::replication`：
  - `peek_changes(cfg) -> Vec<ChangeEvent>`（`pg_logical_slot_peek_binary_changes`，不推进）。
  - `advance_slot(cfg, lsn)`（`pg_replication_slot_advance(slot, lsn)`）。
  - 保留 `pull_changes`（get 语义）但**标注仅供"无持久化"演示/测试**；生产用 peek+advance。
- `engine`：
  - `persist(&Path)`：`text.commit()` + `vector.save` + 写 `checkpoint.json`（原子）。
  - `load(dir, cfg) -> (Engine, applied_lsn)`：开 text + 载 vector + 读 checkpoint。
  - `consume_once(cfg, &mut applier, data) -> usize`：编排 §3.2 的 1→4，返回应用条数。

### 3.4 初始快照 / 流式 衔接（复用 §4）

- **冷启动（无 checkpoint）**：`ensure_slot` 创建逻辑 slot 时 PG 返回一致性点 LSN → `initial_snapshot`（`SELECT *` → 嵌入 → ingest_vector）→ `persist` 且 `applied_lsn = 一致性点` → 之后 peek 增量。
- **流式 CDC（未来）**：把"peek 轮询"换成 `START_REPLICATION` 流，检查点纪律不变——standby 反馈的 flushed LSN 同样**在 persist 之后**才上报。即本计划是流式的前置地基。

## 4. 测试用例（Docker PG，env-gated；纯逻辑部分离线单测）

**离线单测：**
1. `MemVectorIndex::save/load` 往返：内容、维度一致；原子写（写中断不坏旧文件——用临时文件存在性断言）。
2. `checkpoint.json` 序列化往返；dim/model 不符的检测。

**集成（DATABASE_URL，部分 + Ollama/Hash 嵌入）：**
3. **持久化往返**：写 PG → consume_once → 新建 Engine `load` → 向量检索结果一致，**且嵌入调用计数为 0**（用计数包装的 Embedder 验证不重嵌）。
4. **崩溃不丢**（步 3 前崩）：consume 到"已 peek 未 advance"，丢弃 engine 不持久化 → 重启 load 旧 checkpoint → 重 peek 仍见该批 → 应用后数据完整。
5. **崩溃不重**（步 3、4 间崩）：persist 了 checkpoint 但未 advance → 重启 → 重 peek 见同批 → `Applier` 跳过 → 无重复条目。
6. **slot 已推进后**：advance 后再 peek → 空（不重发）。

## 5. 验收标准

- 离线单测 + clippy 净 + fmt 净；集成在 Docker PG（+ Hash/Ollama 嵌入）全绿。
- **杀进程重启**：向量索引不丢、不重嵌、CDC 从检查点续传、无重复/无遗漏（用例 3–6）。
- 13-sync spec 的"崩溃从 slot 续传"表述更正为本计划的精确模型。
- 回写：13-sync（消费语义 + 检查点）、14-engine（persist/load/consume_once）、15-vector（save/load）、看板。
