## 第 ① 条线：本环境就能做（不依赖外部资源，可立即收口）

**特点**：纯 Rust，本机就能跑通验证三绿，不需要数据库/模型/集群。优先做，因为能立刻交付、还能把 GA 出口判据补齐。

|---|---|---|---|

| HNSW filtered-traversal | 带过滤条件搜索时，让"过滤"在遍历图时就生效，而不是搜完一大批再筛 | 带过滤时召回更高、更省 | 过滤场景召回提升，正确性测试通过 |
| ~~CLI 向量落盘~~（已作废） | ~~让命令行版也能把向量存到磁盘~~ → **CLI 已改为 server 的纯 REST 客户端**（2026-06-28），检索/嵌入/落盘归 server，CLI 白嫖全套混合；本项不再需要。见 [CLI 改为 REST 客户端设计](2026-06-28-CLI改为REST客户端设计.md) | — | — |

## 第 ② 条线：真模型多模态 M1（需外部模型服务，价值最高）

**特点**：代码本环境能写 + 写基线测试，但**真验证需要一个多模态 HTTP 模型服务**（图文同空间，如 SigLIP / JinaCLIP 类，经 Ollama 或 OpenAI 兼容端点接入）。文本嵌入已经用 Ollama 验证过（B4），接图像门槛不高。

**这条线解决的核心问题**：把"图像检索 = 搜图注文字"升级成"真的按图像内容检索"。

| 项 | 做什么（大白话） | 做完什么样 |
|---|---|---|
| MM8b | 给 `HttpEmbedder` 接上真图像通道（图片 base64 → 多模态模型 → 向量），并断言"文字和图在同一个向量空间" | 同一张图的向量能被相似图搜到；文图同维同空间 |
| MM9-cross | 真正的"以文搜图 / 以图搜文"（需要模型支持跨模态） | 打一段文字，能召回相关的图 |
| MM10-real | 用真视觉模型给图落向量 + 视频关键帧 + 从对象存储取图字节来嵌入（不只是 inline 小图） | 真图向量召回；大图也能进检索 |
| MM6-signer | 对象存储的真签名 URL（S3 presign 类），安全短链取大媒资 | 大媒资走对象存储签名链，不泄露裸 key |

**建议顺序**：MM8b（钥匙，解锁整条线）→ MM9-cross → MM10-real → MM6-signer。

> **MM6-signer 已细化为正式设计**：[资产网关「控制面 + 两档签名 URL」设计](2026-06-28-MM6-signer-资产网关签名URL设计.md)——把 `/v1/asset` 升级为签发短时签名 URL 的控制面，inline 档（HMAC token，零对象存储）+ object 档（S3 presign），让前端 `<img src>` 免 Bearer 头直接渲染，同时守 ACL / 不当 CDN / 纯 PG 可显图。**注**：inline 档（S1–S3）本环境+Docker 可完整验证，不必等多模态模型——可作为"图像展示"的先行交付，独立于 MM8b。

> **门槛提示**：只要有一个图文同空间的模型服务可达，这条线就能从 `gated` 转成可验证。没有的话，M0 的 caption-as-text 仍独立可用、不影响上线。

## 第 ③ 条线：重型远期（规模大、依赖重，按需启动）

**特点**：面向超大规模视觉文档库和视频精定位，内存/算力开销大，需模型 + 规模验证。**不是上线必需**，等真有这类需求再启动。

| 项 | 做什么（大白话） | 启动前提 |
|---|---|---|
| MM11 | ColPali 多向量 MaxSim 后端（一页图拆成很多 patch 向量，引擎侧算，不进 PG 真源） | 需模型 + 规模 |
| MM12 | ColPali 存储优化：量化 + patch 剪枝 + 先单向量粗筛再多向量精排 | 10 万页级会撑爆纯内存，量化剪枝是前置 |
| MM13 | 视频"时刻检索"：把命中收窄到精确的时间片段边界 | 需模型 |
| parse-vlm | 摄取时用 VLM 给自然图生成语义描述（补图注） | 需 VLM 服务 |

**部署/运维收尾（B7，独立小线）**：容器镜像 build 实测 + docker-compose / CloudNativePG 实跑 + Helm chart + pg 连接池/并发写。需 Docker/K8s 环境，制品大多已写好，差实跑验证。

## 怎么选（决策提示）

- **想立刻有产出、把 GA 补满** → 第 ① 条线。
- **想补上"图像检索"最大的生产缺口** → 第 ② 条线（先 MM8b），但要先有多模态模型服务。
- **有超大视觉库 / 视频精定位的真实需求** → 第 ③ 条线。
- **准备上线部署** → B7 运维收尾。
