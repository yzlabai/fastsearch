## 第 ① 组：本机即可收口（不依赖 DB/模型/集群，优先做）

> 纯 Rust，本机三绿即交付。这组做完，GA 出口判据基本补满。

### 1.1 RaBitQ 旋转接线 engine ✅ —— 复核发现**已完成**

- **复核结论（2026-06-30）**：原条目据 [binary.rs](../../crates/fastsearch-vector/src/binary.rs)/[lib.rs](../../crates/fastsearch-vector/src/lib.rs) 的内联"下一迭代"注释列出，但**代码与 spec 早已完成**：`binary::Rotation`（数据无关固定种子正交变换）已实现，`with_binary_prefilter_rotated` / `set_rabitq_rotation` 启用，engine `open_with` 据 checkpoint `brute_binary_rotated` 翻档接线，server `FASTSEARCH_VECTOR_BACKEND=brute_binary_rotated` 暴露——见 [spec 15 §6](../specs/15-vector.md) 迭代②③（含实测 recall@10 旋转 0.97 vs 0.78）。
- **本次动作**：仅修正两处**陈旧内联注释**（binary.rs:13 / lib.rs:447 的"下一迭代"措辞），使代码↔spec 一致，无功能改动。
- **遗留（真·下一迭代）**：查询分平面量化恢复 popcount 级粗筛速度（spec 已记）。

### 1.2 HNSW 进程内自动压实 ✅ —— 本次落地

- **做什么**：删除/更新只置墓碑（`entries[id]=None`、向量留图中），长跑下 `entries`/图无界增长。新增 `HnswVectorIndex::compact`（活条目原地重建图 + 稠密 id 映射，纯内存）+ `maybe_compact`：删除/更新后「总槽位 > 32 且墓碑过半（dead > live）」自动触发，摊还 O(1)（仿动态数组倍增）；纯新增永不触发、不扰 bulk load。
- **为什么**：无自动压实则高删改场景内存/延迟单调劣化（不变量 #6 墓碑增长项）。
- **验收**：✅ +2 单测（跨阈值后槽位回落=活条目数、墓碑清零；暴力档压实前后 top-k 完全一致）；fmt + clippy -D warnings + test --workspace 三绿。
- **落点**：[hnsw.rs](../../crates/fastsearch-vector/src/hnsw.rs)；已回写 [spec 15 §6](../specs/15-vector.md)。

### 1.3 HTTP Range 支持（媒资体验）

- **做什么**：`/v1/asset/{cid}/bytes` 支持 `Range` 头 → 返回 `206 Partial Content` + `Accept-Ranges`（当前明确未做，[server lib.rs:811](../../crates/fastsearch-server/src/lib.rs#L811)）。先做 inline 档（本机可完整验证），对象存储 Range 随 MM6-object 落地。
- **为什么**：大 inline 媒资断点续传 / 音视频 seek；前端 `<img>/<video>` 友好。
- **验收**：带 `Range: bytes=N-M` 返回 `206` + 正确字节段；越界 416；ACL 仍守（签名 token 门控不被 Range 绕过，进越权用例）。
- **落点**：`fastsearch-server`；回写 [spec 19-server](../specs/19-server.md) + OpenAPI。

### 1.4 CI 矩阵补 OCR/tables 编译档 + 依赖许可清单（GA 出口判据）

- **做什么**：(a) CI 现仅编译校验 parse-ocr/parse-tables（[ci.yml:73](../../.github/workflows/ci.yml)），补一个**显式编译 job** 防 ONNX 档 bit-rot；(b) 接 `cargo-about` / `cargo-deny` 出**依赖许可清单**并设门禁（lindera 中文字典 CC-BY-SA 等需审，见 [词典与第三方许可审](../governance/2026-06-26-词典与第三方许可审.md)）。
- **为什么**：GA 出口判据要求"CI 三档常态绿 + 许可清单可查"（三条线第①线遗留项）；发布前合规。
- **验收**：CI 显式跑 `clippy -p fastsearch-cli --features parse-ocr/--features parse-tables`（编译）；`cargo about generate` 产出清单 artifact，`cargo deny check` 绿。
- **落点**：`.github/workflows/ci.yml` + `deny.toml` + `about.toml`；回写 governance 许可审。

### 1.5 HNSW filtered-traversal（算法，**部分 gated**）

- **做什么**：带过滤搜索时遍历图即过滤、免"搜一大批再筛"（[hnsw.rs:25](../../crates/fastsearch-vector/src/hnsw.rs#L25)）。**前提**：需 `hnsw_rs` 支持遍历期回调过滤——若上游不支持则标 `gated`，本迭代先以 1.2 的自适应过取兜底。
- **为什么**：高选择性过滤场景，后过滤召回塌陷；遍历期过滤是正解。
- **验收**：上游支持→高选择性过滤召回提升、正确性单测过；不支持→在 spec 标 `gated` 并记录上游 issue。
- **落点**：`fastsearch-vector/src/hnsw.rs`；回写 spec 15。

**建议顺序**：~~1.1 旋转接线~~（复核已完成）→ ~~1.2 压实~~（✅ 2026-06-30 落地）→ 1.4 CI/许可（GA 判据，下一项）→ 1.3 Range → 1.5（视上游）。

## 第 ② 组：需外部资源（价值高，半数本机可写测试）

### 2.1 对标基准实跑出数（**最高性价比**）

- **做什么**：[qdrant-adapter](../../benchmarks/qdrant-adapter) + 三件套已就绪，但**还没实跑出对比数**。用标准数据集（如 BEIR 子集 / 自有 KB）跑 fastsearch vs Qdrant：recall@k、p50/p95 延迟、内存。
- **为什么**：差异化叙事（"只需 pgvector + 逻辑复制，不装原生扩展，召回/延迟对标专用向量库"）需要**真实数字**支撑，不能只有架构论证。
- **前提**：本机可起 Qdrant（Docker）+ 一个嵌入服务（Ollama，B4 已验证）。属"需外部资源但门槛低"。
- **验收**：产出一张对比表（recall / 延迟 / 内存，多 ef_search 档）入 [benchmark-strategy](../benchmark-strategy.md)；标清数据集、硬件、嵌入模型；可复现脚本入 `benchmarks/`。
- **落点**：`benchmarks/` + 回写基准策略文档。

### 2.2 pgvector 直查档（"纯 PG 也能查"叙事的闭环）

- **做什么**：向量召回直接走 PG 的 pgvector（HNSW/IVFFlat 索引），作为引擎侧 ANN 之外的**可切换后端**（[vector/lib.rs:7](../../crates/fastsearch-vector/src/lib.rs#L7)，已有 [B6 设计](2026-06-26-B6-pgvector直查档设计.md)）。
- **为什么**：印证"零引擎也能降级查询"，强化"PG 是真源"不变量；小库可省引擎侧内存。
- **前提**：需 `DATABASE_URL` 起 pgvector（CI 已有镜像）。
- **验收**：env-gated 集成测试：同向量两后端 top-K 一致（同分 `GlobalId` tie-break）；**仍只用 `CREATE EXTENSION vector`**，不引原生扩展（守最高优先级不变量）。
- **落点**：`fastsearch-vector` + `fastsearch-pg`；回写 spec 12/15。

### 2.3 server 引擎侧 embedding 写穿（闭合摄取链）

- **做什么**：当前 embedding 须**外部管线先写入 PG**，引擎不自己嵌入（[server main.rs:13/125](../../crates/fastsearch-server/src/main.rs#L13)）。让 server 在 index 路径经 `HttpEmbedder`（Ollama/OpenAI 兼容）写穿向量。
- **为什么**：少一个外部步骤，`/v1/index` 即可端到端；与 B6 CDC 写穿对齐。
- **前提**：需嵌入 HTTP 服务（B4 已验证文本通道）。
- **验收**：配置嵌入端点后 `/v1/index` 自动补向量；无端点时维持现状（外部预写）不报错；e2e（index→search 命中）。
- **落点**：`fastsearch-server` + `fastsearch-embed`；回写 spec 16/19。

### 2.4 MM8b 真图像通道（**最大生产缺口**，gated）

- **做什么**：给 `HttpEmbedder` 接真图像通道（图 base64 → 图文同空间模型 → 向量），断言"文字与图同维同空间"。把线上"图像检索=搜图注文字"升级成真按图像内容检索（三条线第②线钥匙项）。
- **为什么**：当前 caption-as-text 不是真"看图"，这是已知最大生产缺口。
- **前提**：**需图文同空间模型服务**（SigLIP / JinaCLIP 类，经 Ollama 或 OpenAI 兼容端点）。无则 `gated`，M0 caption 链路仍独立可用。
- **验收**：同图向量能被相似图召回；文图同维同空间断言；有服务→`待运行验证`转可验证，无→`gated` + 基线测试就绪。
- **落点**：`fastsearch-embed`；回写 [多模态功能设计](2026-06-25-多模态功能设计与开发计划.md) + spec 16。

**建议顺序**：2.1 对标出数（先做，性价比最高）→ 2.2 pgvector 直查 / 2.3 写穿（闭合链路）→ 2.4 MM8b（待模型服务）。
