## 第 ① 组：本机即可收口（不依赖 DB/模型/集群，优先做）

> 纯 Rust，本机三绿即交付。这组做完，GA 出口判据基本补满。

### 1.1 RaBitQ 旋转接线 engine ✅ —— 复核发现**已完成**

- **复核结论（2026-06-30）**：原条目据 [binary.rs](../../crates/fastsearch-vector/src/binary.rs)/[lib.rs](../../crates/fastsearch-vector/src/lib.rs) 的内联"下一迭代"注释列出，但**代码与 spec 早已完成**：`binary::Rotation`（数据无关固定种子正交变换）已实现，`with_binary_prefilter_rotated` / `set_rabitq_rotation` 启用，engine `open_with` 据 checkpoint `brute_binary_rotated` 翻档接线，server `FASTSEARCH_VECTOR_BACKEND=brute_binary_rotated` 暴露——见 [spec 15 §6](../specs/15-vector.md) 迭代②③（含实测 recall@10 旋转 0.97 vs 0.78）。
- **本次动作**：仅修正两处**陈旧内联注释**（binary.rs:13 / lib.rs:447 的"下一迭代"措辞），使代码↔spec 一致，无功能改动。
- **遗留（真·下一迭代）**：查询分平面量化恢复 popcount 级粗筛速度（spec 已记）。

### 1.2 HNSW 进程内自动压实 ✅ —— 本次落地

- **做什么**：删除/更新只置墓碑（`entries[id]=None`、向量留图中），长跑下 `entries`/图无界增长。新增 `HnswVectorIndex::compact`（活条目原地重建图 + 稠密 id 映射，纯内存）+ `maybe_compact`：删除/更新后「总槽位 > 32 且墓碑过半（dead > live）」自动触发，摊还 O(1)（仿动态数组倍增）；纯新增永不触发、不扰 bulk load。
- **为什么**：无自动压实则高删改场景内存/延迟单调劣化（不变量 #6 墓碑增长项）。
- **验收**：✅ +2 单测（跨阈值后槽位回落=活条目数、墓碑清零；暴力档压实前后 top-k 完全一致）；fmt + clippy -D warnings + test --workspace 三绿。
- **落点**：[hnsw.rs](../../crates/fastsearch-vector/src/hnsw.rs)；已回写 [spec 15 §6](../specs/15-vector.md)。

### 1.3 HTTP Range 支持 ✅ —— 本次落地

- **做什么**：两个 inline 字节出口（authed `GET /v1/asset/{cid}` + token 门控 `/bytes`）支持 `Range`。`parse_range`（单段 `bytes=A-B`/`A-`/`-N`）+ `serve_inline_bytes` 共用：无 Range→200 + `Accept-Ranges: bytes`；命中→206 + `Content-Range`；越界→416 + `bytes */total`；多段→退 200。
- **为什么**：大 inline 媒资断点续传 / 音视频 seek；前端 `<img>/<video>` 友好。
- **验收**：✅ +6 单测（200/206/后缀+开区间/416/多段退化，`serve_inline_bytes` 纯函数确定）；token 门控不变（验签仍在取字节前，Range 不绕过授权）；三绿。**对象存储档 Range** 随 S4 presign（gated）。
- **落点**：[server lib.rs](../../crates/fastsearch-server/src/lib.rs)；已回写 [spec 19-server v1.9](../specs/19-server.md) + OpenAPI（206/416/Range 头）。

### 1.4 CI 矩阵补 OCR/tables 编译档 + 依赖许可清单 ✅ —— 本次落地

- **做什么**：(a) `parse-feature` job 补 `cargo check --features parse-ocr/parse-tables` 两步（本机已验证均编译通过，防 ONNX 档 bit-rot）；(b) 新增 `licenses` job：`cargo deny check licenses sources bans`（[deny.toml](../../deny.toml) allow-list）+ advisories 信息性 + `cargo about generate`（[about.toml](../../about.toml)/[about.hbs](../../about.hbs)）产 `THIRD-PARTY-NOTICES.md` artifact。
- **为什么**：GA 出口判据"CI 三档常态绿 + 许可清单可查"（三条线第①线遗留项）。
- **验收**：✅ cargo-deny 0.19.9 **本机实跑** `cargo deny check licenses sources` → `licenses ok, sources ok`（allow-list 全命中、无冗余）；`cargo metadata` 全树审计无**唯一** copyleft/share-alike 许可；parse-ocr/parse-tables 本机 `cargo check` 均通过。（`cargo about generate` 产清单与 CI job 待 CI 实跑。）
- **落点**：[ci.yml](../../.github/workflows/ci.yml) + deny.toml + about.toml/hbs；已回写 [governance 许可审 §4.3](../governance/2026-06-26-词典与第三方许可审.md)。

### 1.5 HNSW filtered-traversal ✅ —— 本次落地（**非 gated**：上游本就支持）

- **复核结论**：原条目据 spec「需 hnsw_rs 支持」判为部分 gated，**有误**——hnsw_rs 0.3.4 已提供 `search_filter(data, k, ef, Option<&dyn FilterT>)`，`FilterT::hnsw_filter(d_id)` 在层遍历期裁剪（hnsw.rs:1028）。
- **做什么**：有 filter/acl 时把 filter+ACL 谓词下推进遍历（谓词 = 活条目且过 `Filter::eval`+`AclFilter::visible`），结果堆只收合规候选、命中更密；遍历仍穿过被裁节点保连通；自适应过取留作安全网；无 filter/acl 走纯 `search`（零开销）。
- **验收**：✅ +1 测试 `filtered_traversal_acl_no_leak_at_scale`（n=1500>暴力阈值、ACL 谓词，断言无越权泄漏 + 有召回）；既有 filter/acl/recall 测试（now 经 `search_filter`）全过；三绿（224 测试）。
- **落点**：[hnsw.rs](../../crates/fastsearch-vector/src/hnsw.rs)；已回写 [spec 15 §6](../specs/15-vector.md)。

## 第 ② 组：需外部资源（价值高，半数本机可写测试）

### 2.1 对标基准实跑出数（**最高性价比**）

- **做什么**：[qdrant-adapter](../../benchmarks/qdrant-adapter) + 三件套已就绪，但**还没实跑出对比数**。用标准数据集（如 BEIR 子集 / 自有 KB）跑 fastsearch vs Qdrant：recall@k、p50/p95 延迟、内存。
- **为什么**：差异化叙事（"只需 pgvector + 逻辑复制，不装原生扩展，召回/延迟对标专用向量库"）需要**真实数字**支撑，不能只有架构论证。
- **前提**：本机可起 Qdrant（Docker）+ 一个嵌入服务（Ollama，B4 已验证）。属"需外部资源但门槛低"。
- **验收**：产出一张对比表（recall / 延迟 / 内存，多 ef_search 档）入 [benchmark-strategy](../benchmark-strategy.md)；标清数据集、硬件、嵌入模型；可复现脚本入 `benchmarks/`。
- **落点**：`benchmarks/` + 回写基准策略文档。

### 2.2 pgvector 直查档 ✅ —— 复核已实现，本次真机验证 + 并发 boot 加固

- **复核结论（2026-06-30）**：B6 已落地——`PgStore::vector_search`（pgvector ANN + ACL 下推 + filter-aware）、`set_embedding`/`clear_embedding` 写穿、engine `set_pg_vector`/`has_pg_vector`/`vector_pg`、CDC 自动写穿（spec 15 §6 / spec 12 v1.5）。env-gated 集成测试齐备。
- **本次动作**：用 Docker pgvector（cached `pgvector:pg17`）实跑全部 env-gated PG 集成测试——**发现并发 boot 竞态**（`ensure_schema` 的 `CREATE EXTENSION/TABLE/INDEX/PUBLICATION` TOCTOU，`cargo test` 并发下 4/16 非确定失败）→ 修复为**事务级 `pg_advisory_xact_lock` 串行化 DDL**（多副本 boot 安全），+1 回归测试，全 17 集成测试并发多轮全绿。见 [spec 12-pg v1.6](../specs/12-pg.md)。
- **守不变量**：仍只 `CREATE EXTENSION vector` + 逻辑复制，无原生扩展。

### 2.3 server 引擎侧 embedding 写穿 —— 复核：大半已实现，仅 pgvector 直查模式 narrow 剩项

- **复核结论（2026-06-30）**：原条目"引擎不自己嵌入"**不准确**。server 已从 env 配 `HttpEmbedder`（[main.rs:152](../../crates/fastsearch-server/src/main.rs#L152)）并 `set_embedder`；**引擎自带向量后端模式**下 `/v1/index` 已嵌入 passage→写向量索引，CDC `apply_upsert` 路径亦写穿 PG（B6）。真正剩项仅：**pgvector 直查模式**下同步 `/v1/index` 不向 PG `embedding` 列写穿（该模式靠 CDC 或外部预写）。
- **本次验证（真模型 e2e）**：起 server `FASTSEARCH_EMBEDDER=ollama`（`nomic-embed-text-v2-moe` 768d），index 中文 chunk + `mode=vector` 检索——**英文 query 跨语言召回对应中文 chunk 排第一**（量子/股市，零字符重叠），证明 `HttpEmbedder(Ollama)`→engine→向量检索全链路真模型工作。见 [devlog](../devlog/2026-06-30-真机验证-pg并发与ollama语义.md)。
- **剩项落点**：`fastsearch-server`（pgvector 模式同步写穿）+ spec 16/19；非上线必需（CDC 路径已闭合）。

### 2.4 MM8b 真图像通道（**最大生产缺口**，gated）

- **做什么**：给 `HttpEmbedder` 接真图像通道（图 base64 → 图文同空间模型 → 向量），断言"文字与图同维同空间"。把线上"图像检索=搜图注文字"升级成真按图像内容检索（三条线第②线钥匙项）。
- **为什么**：当前 caption-as-text 不是真"看图"，这是已知最大生产缺口。
- **前提**：**需图文同空间模型服务**（SigLIP / JinaCLIP 类，经 Ollama 或 OpenAI 兼容端点）。无则 `gated`，M0 caption 链路仍独立可用。
- **验收**：同图向量能被相似图召回；文图同维同空间断言；有服务→`待运行验证`转可验证，无→`gated` + 基线测试就绪。
- **落点**：`fastsearch-embed`；回写 [多模态功能设计](2026-06-25-多模态功能设计与开发计划.md) + spec 16。

**建议顺序**：2.1 对标出数（先做，性价比最高）→ 2.2 pgvector 直查 / 2.3 写穿（闭合链路）→ 2.4 MM8b（待模型服务）。
