## 1. 不变量与硬门禁（每阶段都守）

1. **检索热路径零 docparse 依赖**（本计划新增的最高门禁）：`fastsearch-core/server/vector/text/engine/embed/rerank/pg/sync` 的依赖图**不得**出现任何 `docparse-*` crate。CI 加 `cargo tree` 断言守住（DF3）。
2. **默认构建精简**：`fastsearch-server` 默认 feature **不含**解析/ONNX/渲染；重件全 opt-in。
3. **PG 真源 / 引擎派生**：解析产物经摄取适配器落 PG（沿现有 `fastsearch index` 写路径），不在引擎侧造权威数据。
4. **ACL 不可绕过**：摄取不引入新检索入口；`tenant`/`acl` 在适配器注入（来自摄取身份/配置），不从解析产物推断越权。
5. **确定性**：解析→chunk 适配确定（docparse 本身字节级确定）；适配器纯函数可单测。
6. **诚实记账 / 重依赖 opt-in**：OCR/渲染/VLM 未验证即标 `下一迭代`；许可未审不分发。
7. **收口**：`cargo fmt --all --check` + `clippy --workspace --all-targets -D warnings` + `cargo test --workspace` 三绿；改契约再实跑 CLI 验证（DEV_SPEC §2）。

## 2. 阶段总览

```
DF1  C 预演：git dep docparse-core + From 适配器 + fastsearch ingest <pdf> 最小打通
DF2  subtree 并入：保历史 + .gitignore 大文件 + workspace 合并去重 + path 依赖
DF3  依赖隔离门禁：CI 断言检索热路径无 docparse-*（最高不变量守护）
DF4  解析器增量并入：docx/html/xlsx/pptx/md/csv/srt/tex/eml/img/adoc（按需）
DF5  feature 门控 + ingest 子命令收敛 face（parse / parse-ocr… 占位）
DF6  CI 矩阵 + 许可审 + 文档收口（CLAUDE 架构图 / 看板 / 职责划分回写）
```

每阶段独立收口、可暂停/回退（DF1–DF2 之间删 git 依赖即恢复；DF2 后 `git revert`/删 `vendor/docparse` 回退，且因热路径零依赖，回退不触检索代码）。

## 4. 测试计划

- **单测（本环境必过）**：`From<docparse::Chunk>` 适配器逐字段映射 + media 指针 + tenant/acl 注入；srt 时间码→`TimeSpan`；各解析器样例往返。
- **依赖隔离（硬门禁）**：DF3.1 的 `cargo tree` 断言——检索热路径零 docparse。
- **默认精简（硬门禁）**：默认 feature 构建不含 tract-onnx/hayro（`cargo tree` 校验）。
- **端到端（gated）**：`fastsearch ingest <pdf>` → PG → `search` 命中（需 PG）。
- **收口**：每阶段 fmt+clippy+test 三绿；改 CLI 契约实跑验证。

## 5. 风险与回退

- **依赖图泄漏**（最高）：docparse crate 不慎被检索热路径传递依赖 → DF3.1 CI 断言兜底，红了即修。
- **workspace 依赖版本冲突**：两仓 `[workspace.dependencies]` 版本不一 → DF2.2 一次性对齐去重；冲突项在 PR 记录取舍。
- **vendored tract 维护负担**：随 `docparse-ocr` 进树 → D-3 默认不纳入验证、仅占位；真正启用前单独评审（承接补丁 + 模型许可）。
- **许可（分发前）**：CC-BY-SA 字典 / ONNX 模型 → DF6.2 审；未审不随二进制分发。
- **face 重复**：docparse 自带 axum/MCP 与 fastsearch 重叠 → DF5.2 收敛为单检索服务 + ingest 子命令。
- **回退**：DF1（git 依赖）删依赖即恢复；DF2+（subtree）`git revert`/删 `vendor/docparse`；**热路径零依赖保证回退不碰检索代码**。
