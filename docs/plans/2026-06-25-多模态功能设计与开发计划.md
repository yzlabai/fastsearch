## 0. 设计要点（TL;DR）

- **不改架构，加字段 + 加路由 + 加一个 ACL 媒资网关**。四个场景（文档图展示 / 图片内容搜 / 语音转录 / 视频片段）全部落在现有 `Chunk + GlobalId + Citation + 混合管线 + CDC` 上。
- **三个关键设计决策**（把需求分析 §6 的开放问题拍成默认值，见 §1）：
  1. **`text` 不 `Option` 化**——语义改为"该 chunk 的可检索文本表示（正文/caption/转录，媒资无文本时为空串）"。避免 6 个 crate 的 ripple，BM25/高亮/rerank 调用点零改动。
  2. **`modality` 是普通可过滤元数据字段，不是新 search 参数**——走现有 `core::Filter` 谓词下推（不变量 #5），零新 API 面、两端预过滤天然一致。
  3. **媒资字节分层 + 统一经 ACL 网关**——小裁图入 PG `bytea`（随逻辑复制走，"一个 PG 即可跑"）；音视频/大图入对象存储，网关 ACL 校验后签短时 URL。
- **里程碑**：**M0(P0)** 派生文本路线，大部分本环境可验证、零新模型依赖 → 交付语音转录检索+音频深链、文档图上下文召回+内联展示、视频转录检索；**M1(P1)** 跨模态单向量，gated on 多模态 HTTP 模型 → 以文/以图搜图、视频关键帧；**M2(P2)** ColPali 多向量 + 视频精定位，重、gated。
- **守的不变量**：托管 PG 只用 `CREATE EXTENSION vector`（多向量 MaxSim 全在引擎侧）、PG 真源/引擎派生、ACL 服务端注入不可绕过、`GlobalId` tie-break、SUPERSET 预过滤 + 精确后过滤、重件 opt-in 诚实记账。

## 1. 设计决策（把需求分析 §6 拍成默认值）

| # | 决策点 | 选定方案 | 理由 / 取舍 |
|---|---|---|---|
| D1 | 媒资字节存哪 | **分层**：小裁图(KB 级)→ PG `bytea`(inline)；音视频/大图→ 对象存储,PG 存 URI | 小图随 CDC 走、保"一个 PG 即可跑"；大媒资不撑爆 WAL/复制。对象存储为**可选依赖**（纯文档场景不需要） |
| D2 | 多模态向量在 PG 怎么存 | **单 `embedding` 列 + 统一多模态模型(同一空间)**；ColPali 多向量**只在引擎派生层,不入 PG** | pgvector 友好、最小改动；多向量违反不变量 #1,故隔离到引擎侧 |
| D3 | 转录/关键帧/caption 谁产出 | **上游 docparse / 媒资预处理服务**产出带时间戳的结构化 chunk；fastsearch 只消费 | 保 fastsearch "检索+溯源"边界；需与 docparse 对齐 chunk schema |
| D4 | query 端跨模态怎么暴露 | **modality 走普通 `Filter` 字段**；**以图搜图 = REST 收图 → 嵌成 query 向量 → 走现有 vector 路径** | 零新检索 API 概念；以图搜图对 core 几乎零改动 |
| D5 | `text` 是否 `Option` 化 | **否**，改语义为"可检索文本表示,可空串" | 避免全仓 ripple（"加字段优先于改语义"）；空串=不进 BM25,正常 |
| D6 | 字节出口安全模型 | **统一 `GET /v1/asset/{cid}` 网关**,复用 `principal→acl_for→可见性`,支持 HTTP Range | ACL 不可绕过（不变量 #3）；Range 支持音视频 seek |

## 2. 数据模型设计（`core` / [model.rs](../../crates/fastsearch-core/src/model.rs)）

### 2.1 ChunkKind + modality
```rust
pub enum ChunkKind { Heading, Paragraph, Table, Code, ListItem, Image, Audio, Video } // 新增 Audio/Video

impl ChunkKind {
    /// 嵌入路由 + 过滤用；作为普通元数据字段下推（D4）。
    pub fn modality(&self) -> Modality { /* Image→Image, Audio/Video→… 其余→Text */ }
}
pub enum Modality { Text, Image, Audio, Video } // serde snake_case，可作 Filter 值
```

### 2.2 Chunk：加 `media`，`text` 语义放宽（D5）
```rust
pub struct Chunk {
    pub text: String,            // 语义放宽：可检索文本表示（正文/caption/转录），媒资无文本时为 ""
    pub kind: ChunkKind,
    // …现有 page/bbox/heading_path/section_id/char_len/tenant/acl 不变…
    pub media: Option<MediaRef>, // 替换原 image_meta（迁移见 §6）
}

pub struct MediaRef {
    pub asset: AssetPointer,            // 如何取到字节
    pub media_type: Option<String>,    // MIME，如 image/png、audio/mpeg、video/mp4
    pub time: Option<TimeSpan>,         // 音视频时间区间（C/D）
    pub region: Option<BBox>,           // 图/关键帧内区域（复用现有 BBox）
    pub caption_source: Option<String>, // 沿用现 ImageMeta 字段
    pub thumbnail: Option<AssetPointer>,// 视频/大图缩略图
}

pub enum AssetPointer {
    Inline,                              // 字节在 PG bytea（小裁图，D1）
    Object { uri: String },              // 对象存储 key/uri（大媒资，D1）
    DocRegion { page: u32, bbox: BBox }, // 仅坐标、无字节：用于"跳转到原文位置"，⚠ 不能直接产出可显示图字节（见 §10 渲染依赖）
}
pub struct TimeSpan { pub start_ms: u64, pub end_ms: u64 }
```

### 2.3 Citation：加时间维 + 媒资获取
```rust
pub struct Citation {
    // …现有 collection/doc_id/chunk_id/page/bbox/heading_path/section_id 不变…
    pub time: Option<TimeSpan>,    // 音视频深链区间
    pub media: Option<MediaRef>,   // 渲染/取字节所需（answer 层据此内联展示）
}
```
`citation_id` 格式与 `to_citation_id`/`Citation::parse` **不变**；时间/媒资走结构字段。`resolve_citation` 是**新增能力**（在 engine/server，见 §6），非改现有函数。

## 3. PG schema 设计（`pg` / [sql.rs](../../crates/fastsearch-pg/src/sql.rs)）

新增列（DDL 仍只 `CREATE EXTENSION vector` + `CREATE PUBLICATION`，**不引任何原生扩展**，守不变量 #1）：

| 列 | 类型 | 用途 |
|---|---|---|
| `modality` | `text` | 过滤下推 + 嵌入路由（D4），由 kind 派生后落列 |
| `media` | `jsonb` | `MediaRef`（不含 inline 字节），替换原 `image_meta` |
| `media_bytes` | `bytea NULL` | 小裁图 inline 字节（D1；`asset=Inline` 时有值） |
| `time_start_ms` | `bigint NULL` | 音视频区间起（C/D），便于 SQL 侧过滤/排序 |
| `time_end_ms` | `bigint NULL` | 音视频区间止 |

- `embedding` 列**不变**（单 `vector(dim)/halfvec(dim)`，D2）。
- 同步改：`ChunkRow` 映射、`insert_sql`/`fetch_doc_sql` 参数、**DDL 单测 golden**（同一 commit，DEV_SPEC §3）。
- `media_bytes` 是大字段：DELETE/replace 走现有 doc 级替换路径即可；逻辑复制只搬小图字节（大媒资在对象存储不进 PG）。

## 4. 嵌入管线设计（`embed` / [fastsearch-embed/src/](../../crates/fastsearch-embed/src/)）

### 4.1 trait 扩展为多模态输入
```rust
pub enum EmbedInput { Text(String), Image(Vec<u8>) } // 音视频走转录文本 = Text；声学嵌入远期

pub trait Embedder {
    fn embed(&self, inputs: &[EmbedInput], kind: EmbedKind) -> Result<Vec<Vec<f32>>>;
    fn caps(&self) -> EmbedCaps; // 支持哪些模态 + 维度 + 是否同一空间
}
```
- **后向兼容**：保留 `embed_text(&[String])` 便捷法（内部包成 `EmbedInput::Text`），现有调用点零改。
- **`HashEmbedder`（确定性基线）**：`Image` 退化为"对 caption/占位 hash 出确定向量"——保 CI/离线/无模型可跑，但**不声称语义**（DEV_SPEC §1）。纯图无 caption → 该路无向量，仅 BM25/不召回。
- **`HttpEmbedder`**：`Image` 走多模态端点（base64 input），对接 SigLIP-2 / JinaCLIP-v2 / jina-v4 / Voyage / Cohere；`caps()` 断言**文本与图同一空间**（D2），否则拒绝跨模态查询。
- **真语义经外部 HTTP，不引进程内推理**（沿用现有决策 / B4）。

## 5. 检索路径设计（`vector` / `text` / `engine`）

### 5.1 vector（[lib.rs](../../crates/fastsearch-vector/src/lib.rs)）
- `VecMeta` 加 `modality`、`time: Option<TimeSpan>`（结果自带，供 Citation 透出）。
- **模态/时间过滤 = `core::Filter` 谓词，零新 Filter 变体**（D4/D6）：`Filter` 是 tuple 变体且按**任意字段名**求值——`Filter::In("modality".into(), vec![FieldValue::Str("image".into()), ...])`、时间区间用 `Filter::Gte("time_start_ms".into(), FieldValue::Int(..))`。**真正的接入点是 `FieldSource` trait**（[filter.rs:43](../../crates/fastsearch-core/src/filter.rs#L43)）：只需让各后端的 `FieldSource::get` 对 `"modality"`/`"time_start_ms"`/`"time_end_ms"` 返回值（Tantivy fast field / PG 行 / `VecMeta`），过滤即"翻译成 SUPERSET 查询 + `Filter::eval` 精确后过滤"，与现有同构、两端自动一致（不变量 #5）。无需新搜索参数、无需改 `Filter` 枚举。
- 单向量路径打分不变；**多向量 MaxSim（P2/ColPali）** 在本 crate 新增 opt-in 后端，与 `MemVectorIndex` 并列（§ M2）。

### 5.2 text（[fastsearch-text/src/](../../crates/fastsearch-text/src/)）
- 复用 `text` 字段灌 caption/转录（D5，不新增列）；`text=""` 时不产 term，正常。
- `modality` 加为 Tantivy fast field（过滤/分面，与 `kind` 并列）。高亮天然适用于转录命中句。

### 5.3 engine（[lib.rs](../../crates/fastsearch-engine/src/lib.rs)）
- **CDC `apply_upsert` 按 modality 路由嵌入**（扩展 B4b 的自动嵌入主循环）：
  - `Text/正文/caption/转录` → 文本嵌入（**M0 即可**）。
  - `Image` 有字节/asset → 图像嵌入（**M1**）；M0 阶段退化为 caption 文本嵌入或不嵌。
  - `Video` → 转录文本嵌入（M0）(+ 关键帧图像嵌入 M1)。
- **`SearchRequest` 加 `query_image: Option<Vec<u8>>`**（以图搜图）：engine 用多模态 embedder 嵌成 `req.vector`，**复用现有 vector 召回**——core 排序/融合零改（D4）。
- `SearchHit` / `Citation` 透出 `modality` / `time` / `media`。
- 融合复用 `fuse`（RRF/归一化），同分 `GlobalId` tie-break 不变（不变量 #4）。

## 6. 溯源与媒资 ACL 网关（`server` / `engine`）—— 本期最大安全面

### 6.1 resolve_citation（engine 新方法）
```rust
// 命中 → 由 citation_id + ACL 解析"如何安全取到这段媒资"
fn resolve_citation(&self, cid: &str, acl: &AclFilter) -> Result<Option<ResolvedAsset>>;
pub struct ResolvedAsset { pub fetch: AssetFetch, pub time: Option<TimeSpan>, pub media_type: Option<String> }
pub enum AssetFetch { InlineBytes(Vec<u8>), SignedUrl{ url:String, expires_s:u64 }, DocRender{ doc_id:String, page:u32, bbox:BBox } }
```
ACL 在 `resolve_citation` 内强制校验：解析出的 chunk 不可见 → 返回 `None`（等同 404），**不暴露存在性**。

### 6.2 REST 端点（[fastsearch-server](../../crates/fastsearch-server/src/)）
- **`GET /v1/asset/{citation_id}`**：`principal_from_headers → acl_for → engine.resolve_citation(cid, &acl)`；
  - `InlineBytes` → 直接吐字节（带 `Content-Type`）；
  - `SignedUrl` → `302` 到短时签名 URL（仅校验通过后签发，**绝不暴露裸 key**）；
  - 支持 **HTTP `Range`**（音视频按 `time` seek/分段回放）。
- **`POST /v1/search` 扩展**：可选 `multipart` 图片字段 → `query_image`（以图搜图）。
- **越权用例进测试**：`asset_acl_not_bypassable`（无权身份取 `/v1/asset/{cid}` 必 404/403），与现有 `acl_not_bypassable` 同级（DEV_SPEC §1）。

## 7. 确定性基线与评测

- **基线可跑**：无多模态模型时，图/音/视频经 caption/转录走文本嵌入 + BM25，全链路 CI 可验证、确定（HashEmbedder + `GlobalId` tie-break）。
- **eval（[fastsearch-eval](../../crates/fastsearch-eval/src/)）**：golden 集扩多模态查询（图 caption 召回、转录命中、跨模态以文搜图），nDCG/recall 入 CI 回归门禁（`assert_no_regression`）。

### 建议执行序与门禁
- **MM0（前置，跨仓）**：与 docparse 锁步敲定共享 chunk schema 扩展（time/audio·video kind/media 指针），见 [职责划分文档 §3](2026-06-25-多模态职责划分-docparse与fastsearch.md)。这是 MM1 的硬前置——两仓字段名/语义须一致，否则 docparse 出的 chunk 喂不进。

## 10. 风险与回退

- **ACL 新出口**（最高）：MM6 网关须先于任何媒资展示，带越权测试；未过不上线。
- **`text` 语义放宽的隐患**：审计所有"假设 `text` 非空"处（BM25/高亮/rerank/snippet）；MM3 专门覆盖空串路径。回退：若空串引发歧义，再评估 `Option` 化（代价见需求分析）。
- **统一多模态模型可得性**（M1）：选定模型需 HTTP 可达且文图同空间；不可得则 M1 整体 `gated`，M0 仍独立交付价值。
- **ColPali 容量**（M2）：10 万页级即击穿纯内存（需求分析 §8），MM12 量化/剪枝/粗筛为前置；未达容量信封不启用。
- **场景 A 的图渲染依赖**（已基本化解，见 [职责划分文档](2026-06-25-多模态职责划分-docparse与fastsearch.md)）：图字节**由 docparse 现成导出**（`--image-embed` base64 → `Inline`、`--image-dir` 文件 → `Object`），fastsearch **不自建 PDF 渲染器**。`DocRegion` 仅作"无导出字节时跳转原文页"的兜底。残留行动：M0 前确认 docparse 导出开关已开 + caption 绑定满足召回。
- **上游耦合**：转录/关键帧质量决定上限；与 docparse 对齐 chunk schema（time/asset 字段、是否裁图落字节）为 M0 前置协调项（D3）。
- **诚实记账**：M1/M2 及 MM2/MM5/MM6 的外部依赖部分，状态一律 `待运行验证`/`gated`，禁写"已完成"。
